    **discovery-inventory-panel.tsx (UI-01):**
    - State: `inventory: ModelInventory | null`, `loading`, `refreshing`
    - On mount: `fetch('/api/discovery/models')` -> setInventory
    - Refresh button: `fetch('/api/discovery/refresh', { method: 'POST' })` then reload, toast.success(t('inventory.refresh'))
    - Render: Card with header showing "Inventario Discovery" + last update timestamp + stale badge if is_stale
    - Provider chips: map providers[] -> Badge green if status==='active', red/gray otherwise, show "{provider} ({model_count})"
    - Models section: group models[] by provider, show as small badges with model.name
    - Empty state: t('inventory.empty')
    - Use i18n keys from settings.modelIntelligence.inventory.*

    **mid-cards-grid.tsx (UI-02):**
    - Props: `models: MidEntry[]`, `onEdit: (m: MidEntry) => void`
    - Also accepts loading state from parent OR fetches itself on mount
    - Group models by tier: Elite / Pro / Libre (order matters)
    - For each tier: h3 header, grid of Cards (3 cols on lg, 2 md, 1 sm)
    - Each card: display_name header + tier Badge (using getTierStyle(tier)), best_use text (line-clamp-2), capabilities as small outline Badges, cost_notes as small text, "Editar" button calling onEdit(model)
    - Skip models where tier is null (or put in "Sin clasificar" group)

    **mid-edit-dialog.tsx (UI-03):**
    - Props: `model: MidEntry | null`, `open: boolean`, `onClose: () => void`, `onSaved: (updated: MidEntry) => void`
    - Use shadcn Dialog. Form state initialized from `model` when it changes.
    - Form fields:
      * display_name (Input)
      * tier (Select: Elite / Pro / Libre / null)
      * best_use (Textarea, 3 rows)
      * capabilities (tag editor: Input with onKeyDown Enter pushes trimmed value to array; display current as Badges with X button to remove)
      * cost_notes (Input)
      * scores: 5 Sliders for reasoning, coding, creativity, speed, multilingual (0-10, step 1); show current value next to label
      * status (Switch: active/retired)
    - Save button: PATCH /api/mid/{model.id} with diff, on success toast.success(t('mid.saved')), call onSaved, close. On failure toast.error(t('mid.saveError'))
    - Cancel button closes without saving

    Reference component patterns: look at existing dialogs in `app/src/components/` (any *-dialog.tsx) for shadcn Dialog usage. Discovery panel refresh pattern is in RESEARCH.md.
