Close UI-06 gap: CatBot recommendation Aplicar/Ignorar buttons never render inline because the tool emitter and the UI parse guard disagree on payload shape.

**Root cause confirmed (from code inspection):**
- `recommend_model_for_task` tool (app/src/lib/services/catbot-tools.ts:2314-2333) emits: `{ recommended: { model_key, display_name, tier, provider, best_use, reason }, alternatives: [...], warning? }` — NO `alias_target`, NO top-level `recommended_model`, NO `justification` field.

**Fix strategy:** Align the tool emitter to produce the three fields the UI guard needs (surgical, keeps ModelRecommendationActions props untouched and the downstream PATCH contract intact).

Purpose: User can click Aplicar inline in CatBot chat instead of navigating to Settings > Routing.
Output: Tool emits `alias_target` + `recommended_model` + `justification` at the top level (while preserving existing fields for backward compatibility), parse guard aligned defensively to accept either shape.

    6. Replace `parseRecommendation` at line ~87-108 with a version that accepts BOTH shapes (new flat + legacy nested) so it stays robust if the tool shape ever shifts again:
       ```typescript
       function parseRecommendation(
         result: unknown,
       ): { recommended_model: string; alias_target: string; justification: string } | null {
         if (!result) return null;
         let obj: unknown = result;
         if (typeof obj === 'string') {
           try { obj = JSON.parse(obj); } catch { return null; }
         }
         if (!obj || typeof obj !== 'object') return null;
         const r = obj as Record<string, unknown>;

         // Prefer flat shape
         let recommended_model = typeof r.recommended_model === 'string' ? r.recommended_model : null;
         const alias_target = typeof r.alias_target === 'string' ? r.alias_target : null;
         let justification = typeof r.justification === 'string' ? r.justification : null;

         // Fallback: derive from legacy nested shape
         if (!recommended_model && r.recommended && typeof r.recommended === 'object') {
           const nested = r.recommended as Record<string, unknown>;
           if (typeof nested.model_key === 'string') recommended_model = nested.model_key;
           if (!justification && typeof nested.reason === 'string') justification = nested.reason;
         }

         if (!recommended_model || !alias_target || !justification) return null;
         return { recommended_model, alias_target, justification };
       }
       ```
       (Note: if only the flat `alias_target` is missing the guard returns null, which is the intended behavior — without an alias target the Aplicar button has nothing to PATCH.)
