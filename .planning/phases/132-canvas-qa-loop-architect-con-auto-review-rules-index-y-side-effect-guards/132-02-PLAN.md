Contract semantics:
- Returns ArchitectDesign on accept (iter 0 or iter 1)
- Returns the design directly if it contains non-empty needs_cat_paws (short-circuit)
- If the architect response contains `needs_rule_details: string[]` (non-empty), runArchitectQALoop performs an intra-iteration expansion pass: calls getCanvasRule() for each id, appends the expanded detail to the architect user input under "expanded_rules", re-calls the architect once, and then proceeds with QA on the re-called result. The expansion pass does NOT consume one of the 2 QA iterations; it happens before QA within the same iteration slot.
- Returns null after marking job failed + logKnowledgeGap when loop exhausted

Type for QA response (used in tests):
```typescript
interface QaReport {
  quality_score?: number;
  issues?: Array<{ severity: 'blocker'|'major'|'minor'; rule_id: string; node_id: string; description: string; fix_hint: string }>;
  data_contract_analysis?: Record<string, string>;
  recommendation?: 'accept' | 'revise' | 'reject';
}
```
