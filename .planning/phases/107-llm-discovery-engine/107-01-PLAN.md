  <action>
Create `app/src/lib/services/discovery.ts`:

1. **Types** at top of file:
   - `DiscoveredModel` interface: `id` (unique: "provider/model_id"), `name` (display), `provider` ('ollama'|'openai'|'anthropic'|'google'|'litellm'), `model_id` (raw), `is_local` (bool), `size_mb` (number|null), `parameter_size` (string|null), `family` (string|null), `quantization` (string|null), `is_embedding` (bool), `modified_at` (string|null)
   - `ProviderStatus` interface: `provider`, `status` ('connected'|'disconnected'|'no_key'), `latency_ms` (number|null), `error` (string|null), `model_count` (number)
   - `ModelInventory` interface: `models` (DiscoveredModel[]), `providers` (ProviderStatus[]), `cached_at` (ISO string), `ttl_ms` (number), `is_stale` (bool)

2. **Constants**: `CACHE_KEY = 'discovery:inventory'`, `CACHE_TTL = 300_000` (5 min)

3. **discoverOllama()**: Fetch `${OLLAMA_URL}/api/tags` with 5s timeout. Map each model to DiscoveredModel. Extract `modified_at` from the raw Ollama response (the field is present in each model object). Use `ollama.ts` pattern for family/parameter_size from `details`. Wrap in try-catch, return empty array on failure.

4. **discoverProvider(provider, endpoint, key)**: Per-provider fetch logic reusing exact patterns from the test route:
   - OpenAI: GET `${endpoint}/models` with `Authorization: Bearer ${key}`. Map `data.data[].id` to DiscoveredModel with provider='openai'. Filter to chat models (starts with 'gpt-' or 'o' or 'chatgpt').
   - Anthropic: GET `${endpoint}/models` with `x-api-key` + `anthropic-version: 2023-06-01`. Map `data.data[].id`. If /models fails, return known fallback list is NOT acceptable -- return empty (DISC-07).
   - Google: GET `${endpoint}/models?key=${key}`. Map `data.models[].name` (strip 'models/' prefix). Filter to 'gemini' models.
   All with AbortSignal.timeout(10000). Wrap each in try-catch, return empty on failure.

5. **discoverLiteLLM()**: Fetch LiteLLM `/v1/models` using existing litellm.ts pattern. Mark models with provider='litellm'. Deduplicate: if same model_id exists from a direct provider, skip the litellm version.

6. **discoverAll()**: Read active providers from `api_keys` table. Build array of discovery promises (always include Ollama + LiteLLM). Use `Promise.allSettled()`. Collect models + provider statuses. Track latency per provider (Date.now() before/after). Return ModelInventory.

7. **getInventory(forceRefresh?)**: Check `cacheGet<ModelInventory>(CACHE_KEY)`. If cached and not forceRefresh, return it. Otherwise call `discoverAll()`, `cacheSet()`, return result. Start background refresh with `setInterval` after first successful discovery (unref the timer). NO top-level side effects -- everything lazy.

8. **inventoryToMarkdown(inv)**: Group models by provider. Format as readable markdown with provider status headers and model bullets showing name, model_id, parameter_size, local/API, chat/embedding. Include cached_at timestamp.

Export all types and `getInventory`, `inventoryToMarkdown`, `discoverAll` (for testing).
