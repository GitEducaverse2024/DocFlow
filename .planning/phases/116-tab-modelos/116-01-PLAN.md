3. **Create TabModelos** at `src/components/settings/model-center/tab-modelos.tsx`:
   - 'use client' component
   - On mount, fetch `/api/mid` and `/api/aliases` in parallel with `Promise.all`
   - Build usageMap: `Map<string, string[]>` mapping model_key to alias names. **IMPORTANT pitfall:** alias `model_key` may not directly match MID `model_key` (alias uses LiteLLM names like `gemini-main`, MID uses `google/gemini-2.5-pro`). To handle: for each alias, check if any MID entry's `model_key` ends with or contains the alias `model_key`, or if they match exactly. Log mismatches to console for debugging.
   - Define extended type locally: `interface MidEntryExt extends MidEntry { auto_created?: number }`
   - Filter state managed with useState, passed to TabModelosFilters
   - Apply filters client-side: tier filter, provider filter, enUsoOnly (check usageMap has entries for model_key)
   - Group filtered models by tier using same pattern as MidCardsGrid: Elite, Pro, Libre sections + "Sin clasificar" section
   - "Sin clasificar" logic: `auto_created === 1 AND (best_use starts with 'Auto-detectado' OR tier is null)`
   - Each tier section: heading with tier name + Badge count, then grid of cards (3 cols on lg, 2 on md, 1 on sm)
   - Each card renders: display_name, provider badge, tier badge (using getTierStyle), best_use text, cost_tier if present
   - "En uso" badge: if model_key is in usageMap, show green Badge with alias count and alias names as tooltip or comma-separated
   - Include Edit button (Pencil icon) on each card that opens MidEditDialog (import from existing `@/components/settings/mid-edit-dialog`)
   - Loading state with skeleton cards while fetching
   - Error state with toast (sonner) on fetch failure
