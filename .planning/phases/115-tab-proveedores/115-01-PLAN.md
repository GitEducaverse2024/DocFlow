  <action>
Create `TabProveedores` component in `app/src/components/settings/model-center/tab-proveedores.tsx`.

**Data fetching (on mount):**
- Fetch GET `/api/settings/api-keys` for provider configs (ProviderConfig[])
- Fetch GET `/api/models/health` for provider health status (HealthResult)
- Use useState + useEffect pattern (same as tab-resumen.tsx)
- Show Loader2 spinner while loading

**Define locally (do NOT import from page.tsx):**
- `ProviderConfig` interface (same shape as page.tsx lines 19-28)
- `PROVIDER_META` record (same data as page.tsx lines 30-36)
- `ProviderHealth` and `HealthResult` interfaces (same as tab-resumen.tsx lines 13-36, only providers + checked_at + cached needed)

**Collapsed card (default state):**
- Each provider renders as a Card with onClick to expand
- Show: emoji (from PROVIDER_META) + provider name + semaphore dot (green=connected from health data, red=error, gray=no health data) + model_count badge + chevron icon
- Semaphore: 8px circle, use `bg-emerald-500` for connected, `bg-red-500` for error, `bg-zinc-600` for unknown
- Compact: single row, ~48px height, cursor-pointer, hover:border-violet-800/30
- Dark theme: bg-zinc-900/80 border border-zinc-800

**Accordion behavior:**
- State: `expandedProvider: string | null` (null = all collapsed)
- Clicking a collapsed card sets expandedProvider to that provider key
- Clicking the expanded card collapses it (set null)
- CSS transition: max-height with overflow-hidden, duration-200

**Expanded card content (when expandedProvider matches):**
- **API Key section** (only if meta.needsKey):
  - If has_key: show masked key (readonly Input type=password) + Eye/EyeOff toggle + Trash2 delete button
  - If !has_key: show Input for new key + "Guardar" button (violet gradient)
  - Delete: show inline confirm ("Seguro?" with Confirmar/Cancelar) before calling DELETE endpoint
- **Endpoint section:**
  - Show current endpoint as clickable text (font-mono, underline dotted)
  - Click to enter edit mode: Input + OK + Cancel buttons
  - Save calls PATCH with { endpoint }
- **Test button ("Probar"):**
  - Button with FlaskConical icon, calls POST `.../test`
  - While testing: Loader2 spinner
  - On success: toast.success, show model count badge, update provider's local test_status to 'ok'
  - On error: toast.error with message, update test_status to 'failed'
- **Auto-test on save:** After successful API key save (PATCH), automatically trigger the test endpoint. Flow: Save key -> on success -> run test -> update semaphore
- **Test result display:** If test returned models, show count as small Badge below test button (e.g., "12 modelos disponibles"). If error, show error text in red.
- **Last tested:** Show `last_tested` timestamp if available

**Refresh callback:**
- `fetchProviders()` function to re-fetch api-keys after any mutation (save/delete/test)
- Also re-fetch health data after test completes to update semaphore

**i18n:** Use `useTranslations('settings.modelCenter.proveedores')` for all strings. Keys will be added in Task 2.

**Imports:** Use shadcn Card/CardContent, Badge, Button, Input from @/components/ui/*. Icons from lucide-react: Eye, EyeOff, Trash2, FlaskConical, Loader2, ChevronDown, ChevronRight. Toast from sonner.

**Style conventions (from CLAUDE.md/skills):**
- bg-zinc-900/80 border border-zinc-800 for cards
- hover:border-violet-800/30 for interactive
- bg-gradient-to-r from-violet-600 to-purple-700 for primary buttons
- text-zinc-50 for headings, text-zinc-400 for secondary, text-zinc-500 for labels
- Font mono for API keys and endpoints
