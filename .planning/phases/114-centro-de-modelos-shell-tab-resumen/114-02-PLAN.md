```typescript
export interface ProviderHealth {
  provider: string;
  status: 'connected' | 'error';
  latency_ms: number | null;
  model_count: number;
  error: string | null;
}

export interface AliasHealth {
  alias: string;
  configured_model: string;
  resolved_model: string | null;
  resolution_status: 'direct' | 'fallback' | 'error';
  error: string | null;
  latency_ms: number;
}

export interface HealthResult {
  providers: ProviderHealth[];
  aliases: AliasHealth[];
  checked_at: string;
  cached: boolean;
}
```

API endpoints available:
- GET /api/models/health?force=true — returns HealthResult
- POST /api/discovery/refresh — triggers full discovery scan

    2. Uses `useTranslations('settings.modelCenter.resumen')` for all strings
    3. State: `healthData: HealthResult | null`, `loading: boolean`, `verifying: boolean`
    4. On mount: fetch GET /api/models/health (no force) to show cached data quickly
    5. Types: Import ProviderHealth, AliasHealth, HealthResult from `@/lib/services/health` (these are the type definitions, safe for client import since they're just interfaces)
       - Actually, do NOT import from server module. Duplicate the interfaces locally in the component or create a shared types file. Since health.ts imports server-only modules (cache, discovery, alias-routing), importing it in a client component will fail.
       - Best approach: Define the HealthResult/ProviderHealth/AliasHealth types inline at the top of tab-resumen.tsx (or create a `app/src/types/health.ts` shared types file).

    **Layout (two sections):**

    A) **Providers section** — Card with title "Proveedores"
       - Grid of mini-cards (2 cols on md, 1 on mobile)
       - Each provider card shows:
         - Circle semaphore: emerald-500 for connected, red-500 for error
         - Provider name (bold)
         - Latency badge: "{latency_ms}ms" or "—" if null
         - Model count: "{model_count} modelos"
         - If error: small red text with error message

    B) **Aliases section** — Card with title "Aliases"
       - Grid of mini-cards (2 cols on md, 1 on mobile)
       - Each alias card shows:
         - Circle semaphore: emerald-500 for direct, amber-500 for fallback, red-500 for error
         - Alias name (bold, font-mono)
         - Resolution status badge: "Directo", "Fallback", "Error"
         - Resolved model (font-mono, text-xs): the actual model resolved
         - If fallback: show configured_model -> resolved_model
         - If error: show error message in red text

    **Top bar (above sections):**
    - "Ultimo check: hace X min" computed from checked_at using relative time
      - Use simple relative time: compute diff in seconds, show "hace Xs", "hace Xmin", "hace Xh"
      - Update every 30s via setInterval
    - "Verificar" button (right-aligned):
      - On click, sets verifying=true
      - Calls in sequence: POST /api/discovery/refresh, then POST /api/mid/sync, then GET /api/models/health?force=true
      - On complete: updates healthData, sets verifying=false
      - Shows Loader2 spinner while verifying
      - On error: toast.error with message
    - If healthData.cached is true, show subtle "(cache)" badge next to timestamp

    **Empty/loading states:**
    - While loading: centered Loader2 spinner
    - If no data after load: "Sin datos de salud. Pulsa Verificar para ejecutar el primer check."

    **Conventions:**
    - Use `process['env']` bracket notation if any env vars needed (none expected here)
    - Use sonner toast for error feedback
    - All visible text via t() translations
