Create a new component `GmailWizard` as a Dialog-based 4-step wizard. Follow the canvas-wizard.tsx pattern for step management.

**Props:**
```typescript
interface GmailWizardProps {
  open: boolean;
  onClose: () => void;
  onCreated: () => void; // callback to refresh connectors list
}
```

**State:**
- step: 1 | 2 | 3 | 4
- accountType: 'personal' | 'workspace'
- authMode: 'app_password' | 'oauth2'
- Form fields: name, email, appPassword, fromName, domain (workspace), clientId, clientSecret, authCode
- OAuth2 state: authUrl, exchangeResult (refresh_token_encrypted, client_secret_encrypted)
- Test state: testPhase (0-3), testResults array of { label, status: 'pending'|'running'|'ok'|'error', message? }
- saving: boolean

**Step 1 — Tipo de cuenta (UI-03):**
- Two clickable cards side by side:
  - Gmail Personal: icon Mail, "Gmail Personal", "Usa App Password con tu cuenta personal @gmail.com", emerald border on select
  - Google Workspace: icon Building2, "Google Workspace", "Cuenta corporativa con dominio propio. App Password o OAuth2.", emerald border on select
- "Siguiente" button (disabled until selection)
- Back button hidden on step 1

**Step 2A — App Password Personal (UI-04):**
- Shown when accountType='personal' AND authMode='app_password'
- Fields: Nombre remitente (text), Email Gmail (email), App Password (password type, placeholder "xxxx xxxx xxxx xxxx")
- Info box with link: "Necesitas una App Password de Google. [Como obtenerla](https://myaccount.google.com/apppasswords)"
- Siguiente + Atras buttons

**Step 2B — App Password Workspace (UI-05):**
- Shown when accountType='workspace' AND authMode='app_password'
- Same fields as 2A plus: Dominio (text, placeholder "empresa.com")
- Note: "Workspace usa smtp-relay.gmail.com para envios desde el dominio"
- Toggle link at bottom: "Usar OAuth2 en lugar de App Password" (switches authMode to 'oauth2')

**Step 2C — OAuth2 Workspace (UI-06):**
- Shown when accountType='workspace' AND authMode='oauth2'
- Fields: Nombre remitente, Email, Client ID, Client Secret (password type)
- "Generar URL de Autorizacion" button: calls GET /api/connectors/gmail/oauth2/auth-url with clientId and clientSecret query params
- On success: shows authUrl in a readonly textarea with a "Copiar" button
- Instruction text: "1. Abre la URL en tu navegador. 2. Autoriza la aplicacion. 3. Copia el codigo que aparece y pegalo aqui."
- Textarea for authorization code
- "Intercambiar Codigo" button: calls POST /api/connectors/gmail/oauth2/exchange-code with { code, client_id, client_secret }
- On success: store encrypted tokens, show green checkmark, enable Siguiente
- On error: show error message inline
- Collapsible instructions section (OAUTH-05): "Como configurar Google Cloud Console" with steps: crear proyecto, habilitar Gmail API, crear credencial OAuth2, configurar pantalla de consentimiento, agregar redirect URI OOB
- Toggle link: "Usar App Password en lugar de OAuth2"

**Step 3 — Test de conexion (UI-07):**
- 3 status lines with animated indicators:
  1. "Conectando al servidor SMTP..." -> "Conexion SMTP establecida" or error
  2. "Verificando autenticacion..." -> "Autenticacion correcta" or error
  3. "Enviando email de prueba..." -> "Email de prueba enviado" or error
- Each line has: Loader2 spinning (running), CheckCircle2 green (ok), XCircle red (error)
- Auto-starts on mount:
  - For app_password: call POST /api/connectors/gmail/test-credentials with { user, app_password, account_type, auth_mode: 'app_password' }
  - Simulate 3 phases with 800ms delays between status line updates
  - If test-credentials returns ok, then call POST /api/connectors/gmail/send-test-email
  - For oauth2: only test SMTP/auth (test-credentials with oauth2 params), skip send-test since tokens are pre-validated by exchange
- "Reintentar" button if any phase fails
- "Siguiente" enabled only if all 3 pass (or 2 for oauth2)
- "Omitir test" link (skips to step 4 without testing)

**Step 4 — Confirmacion (UI-08):**
- Summary card with:
  - Type: Gmail Personal / Google Workspace (App Password) / Google Workspace (OAuth2)
  - Email: configured address
  - Remitente: from_name
  - Badge: "Listo para usar" in emerald
- Usage snippets section:
  - Canvas: "Agrega un nodo Conector Gmail al final de tu flujo"
  - Tareas: "Selecciona este conector en un paso de tarea"
  - CatBot: "Pide a CatBot: 'Envia un email usando [nombre]'"
- "Crear Conector" button:
  - Calls POST /api/connectors with:
    - name: fromName or email
    - type: 'gmail'
    - emoji: envelope icon
    - gmail_subtype: accountType === 'personal' ? 'gmail_personal' : (authMode === 'oauth2' ? 'gmail_workspace_oauth2' : 'gmail_workspace')
    - config: JSON.stringify of the appropriate GmailConfig fields (user, account_type, auth_mode, app_password OR client_id/client_secret/refresh_token encrypted values, from_name, domain if workspace)
  - On success: toast.success, call onCreated(), close wizard
  - On error: toast.error with message

**Progress bar:**

**Styling:**
