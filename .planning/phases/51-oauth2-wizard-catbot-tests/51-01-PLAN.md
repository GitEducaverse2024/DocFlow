  <action>
Create two new Next.js API routes:

**GET /api/connectors/gmail/oauth2/auth-url:**
- Accept query params: client_id (required), client_secret (required)
- Create google.auth.OAuth2 client from googleapis package (already installed in Phase 50)
- Generate authorization URL with:
  - redirect_uri: 'urn:ietf:wg:oauth:2.0:oob'
  - scope: 'https://mail.google.com/'
  - access_type: 'offline'
  - prompt: 'consent' (to always get refresh_token)
- Return JSON: { url: string }
- On error return 400/500 with { error: string }
- Export `dynamic = 'force-dynamic'` (reads no env vars but has dynamic params)

**POST /api/connectors/gmail/oauth2/exchange-code:**
- Accept JSON body: { code, client_id, client_secret } (all required)
- Create google.auth.OAuth2 client with client_id and client_secret
- Set redirect_uri to 'urn:ietf:wg:oauth:2.0:oob'
- Call oauth2Client.getToken(code) to exchange for tokens
- Extract refresh_token from response
- If no refresh_token returned, return 400 error "No se obtuvo refresh_token. Intenta revocar acceso en myaccount.google.com y reintentar."
- Encrypt refresh_token using encrypt() from crypto.ts
- Encrypt client_secret using encrypt() from crypto.ts
- Return JSON: { refresh_token_encrypted, client_secret_encrypted, client_id } (NEVER return raw client_secret)
- On error return appropriate status with { error: string } using translateError pattern
- Export `dynamic = 'force-dynamic'`

Both routes: use try/catch, log with logger, validate required params returning 400 if missing.
