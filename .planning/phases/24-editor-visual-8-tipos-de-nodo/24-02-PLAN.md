  <action>
Create `app/src/components/canvas/node-config-panel.tsx` — a single "use client" component that renders at the bottom of the editor and shows type-specific configuration forms for the selected node.

**Panel behavior:**
- Auto-opens when a node is selected (selectedNodes.length > 0), auto-closes when selection is cleared
- Collapsible: toggle button to collapse/expand, remembers collapsed state
- Position: fixed at bottom, overlaying the canvas. Height ~250px when expanded, ~40px collapsed (just the toggle bar)
- Styling: `bg-zinc-900 border-t border-zinc-800 shadow-lg` with `z-20`
- Header shows: node type icon + label + node name, collapse toggle (ChevronDown/ChevronUp)

**Props:** Receive selectedNode (Node | null) and a callback for updating node data.

**Integration pattern:** In canvas-editor.tsx (CanvasShell):
- Track selected node: use `onSelectionChange` or `onNodeClick` to capture the selected node
- Pass selected node to NodeConfigPanel
- NodeConfigPanel calls `useReactFlow().updateNode(nodeId, { data: { ...existingData, ...changes } })` to persist form changes to React Flow state (which triggers auto-save in Plan 03)

**Per-type forms (all form labels in Spanish):**

**START config:** Simple textarea "Input inicial" (optional, maps to data.initialInput)

**AGENT config (NODE-02):**
- Agent selector: `<select>` populated by fetching GET /api/agents on panel mount. Maps to data.agentId. Show agent name.
- Model override: `<input>` text for data.model (optional, placeholder "Modelo del agente")
- Instructions: `<textarea>` for data.instructions
- RAG toggle: `<input type="checkbox">` for data.useRag with label "Usar RAG"
- Skills selector: multi-select or checkboxes populated by fetching GET /api/skills. Maps to data.skills array

**PROJECT config (NODE-03):**
- Project selector: `<select>` populated by fetching GET /api/projects. Maps to data.projectId
- RAG Query: `<textarea>` for data.ragQuery (placeholder "Consulta RAG")
- Max chunks: `<input type="number" min="1" max="20">` for data.maxChunks (default 5)

**CONNECTOR config (NODE-04):**
- Connector selector: `<select>` populated by fetching GET /api/connectors. Maps to data.connectorId
- Mode: `<select>` with options "before" and "after". Maps to data.mode
- Payload template: `<textarea>` for data.payload (placeholder "Plantilla de payload JSON")

**CHECKPOINT config (NODE-05):**
- Instructions: `<textarea>` for data.instructions (placeholder "Instrucciones para el revisor")

**MERGE config (NODE-06):**
- Synthesizer agent: `<select>` populated from GET /api/agents (optional). Maps to data.agentId
- Instructions: `<textarea>` for data.instructions (placeholder "Instrucciones de sintesis")
- Handle count: number input (2-5) for data.handleCount

**CONDITION config (NODE-07):**
- Condition: `<textarea>` for data.condition (placeholder "Condicion en lenguaje natural, ej: 'El documento tiene mas de 500 palabras'")

**OUTPUT config (NODE-08):**
- Output name: `<input>` for data.outputName (default "Resultado")
- Format: `<select>` with "markdown", "json", "plain". Maps to data.format

**Fetch pattern for selectors:** Use `useEffect` with empty deps to fetch agents/projects/connectors/skills lists on panel mount. Store in local state. Only fetch what the current node type needs (conditional fetch based on selectedNode.type).

**Important:** The panel is ONE component with conditional rendering per node type — NOT separate files per form. Keep it manageable by using helper sub-functions (renderAgentForm, renderProjectForm, etc.) within the same file.

Wire the NodeConfigPanel into canvas-editor.tsx's CanvasShell, positioned after the canvas container div inside the flex column layout.
