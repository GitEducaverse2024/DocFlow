# Phase 78-01: Employee Timesheets Fix

## Goal
Las fichas de jornada se crean/actualizan con timestamps Unix correctos respetando timezone Europe/Madrid. Los tools aceptan date + HH:MM como entrada y convierten server-side.

## Requirements Covered
TFIX-01, TFIX-02, TFIX-03, TFIX-04

## Context
- **Repo**: `~/holded-mcp/`
- **Bug**: `holded_create_timesheet` y `holded_update_timesheet` requieren Unix timestamps como input. El LLM no puede hacer conversion timezone de forma fiable, causando timestamps incorrectos.
- **Fix**: Cambiar inputs a `date` (YYYY-MM-DD) + `startTime`/`endTime` (HH:MM). Conversion a Unix timestamp server-side con Europe/Madrid.

### Task 1: Add `timeToTimestamp()` to date-helpers.ts
**File**: `src/utils/date-helpers.ts`
**Action**: Add function

Add a timezone-aware conversion function:

```typescript
/**
 * Convert date string (YYYY-MM-DD) + time string (HH:MM) to Unix timestamp (seconds)
 * using the specified IANA timezone.
 *
 * Uses Node.js Intl API for correct CET/CEST offset detection.
 */
export function timeToTimestamp(dateStr: string, timeStr: string, tz: string = 'Europe/Madrid'): number {
  const [year, month, day] = dateStr.split('-').map(Number);
  const [hours, minutes] = timeStr.split(':').map(Number);

  // Create a Date in UTC with the wall-clock values
  const naiveUtcMs = Date.UTC(year, month - 1, day, hours, minutes, 0, 0);

  // Determine the UTC offset for this timezone at this approximate time
  // by comparing how the same instant renders in UTC vs the target timezone
  const probeDate = new Date(naiveUtcMs);
  const utcParts = getDateParts(probeDate, 'UTC');
  const tzParts = getDateParts(probeDate, tz);

  // Offset in ms = tz_representation - utc_representation
  const utcMs = Date.UTC(utcParts.year, utcParts.month - 1, utcParts.day, utcParts.hour, utcParts.minute);
  const tzMs = Date.UTC(tzParts.year, tzParts.month - 1, tzParts.day, tzParts.hour, tzParts.minute);
  const offsetMs = tzMs - utcMs;

  // The user wants `hours:minutes` in the target timezone
  // So the UTC time = naiveUtcMs - offsetMs
  const targetUtcMs = naiveUtcMs - offsetMs;

  return Math.floor(targetUtcMs / 1000);
}

function getDateParts(date: Date, tz: string): { year: number; month: number; day: number; hour: number; minute: number } {
  const formatter = new Intl.DateTimeFormat('en-US', {
    timeZone: tz,
    year: 'numeric',
    month: 'numeric',
    day: 'numeric',
    hour: 'numeric',
    minute: 'numeric',
    hour12: false,
  });
  const parts = formatter.formatToParts(date);
  return {
    year: Number(parts.find(p => p.type === 'year')!.value),
    month: Number(parts.find(p => p.type === 'month')!.value),
    day: Number(parts.find(p => p.type === 'day')!.value),
    hour: Number(parts.find(p => p.type === 'hour')!.value),
    minute: Number(parts.find(p => p.type === 'minute')!.value),
  };
}
```

#### date-helpers.test.ts — Test `timeToTimestamp()`
```typescript
describe('timeToTimestamp', () => {
  it('converts CET date+time correctly (winter)', () => {
    // 2026-03-17 09:00 CET (UTC+1) = 2026-03-17 08:00 UTC = 1773734400
    expect(timeToTimestamp('2026-03-17', '09:00')).toBe(1773734400);
  });

  it('converts CEST date+time correctly (summer)', () => {
    // 2026-06-15 09:00 CEST (UTC+2) = 2026-06-15 07:00 UTC = 1781506800
    expect(timeToTimestamp('2026-06-15', '09:00')).toBe(1781506800);
  });

  it('converts end-of-day time', () => {
    // 2026-03-17 18:00 CET (UTC+1) = 2026-03-17 17:00 UTC
    const expected = Math.floor(new Date('2026-03-17T17:00:00Z').getTime() / 1000);
    expect(timeToTimestamp('2026-03-17', '18:00')).toBe(expected);
  });

  it('handles midnight correctly', () => {
    // 2026-03-17 00:00 CET (UTC+1) = 2026-03-16 23:00 UTC
    const expected = Math.floor(new Date('2026-03-16T23:00:00Z').getTime() / 1000);
    expect(timeToTimestamp('2026-03-17', '00:00')).toBe(expected);
  });
});
```
