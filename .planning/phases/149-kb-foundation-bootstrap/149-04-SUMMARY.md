## Decisions Made

1. **Vanilla Node `.cjs` over TypeScript compilation or ts-node**: Repo root has no `package.json`, so the CLI must run standalone. Duplicated YAML parser (~200 lines) from `validate-kb.cjs` is the cost. Upgrade path documented: add AJV + js-yaml to a future root package.json, unify parsers across three files.
2. **Serializer key order via ORDER[] constant**: Schema-mandated key order in frontmatter; a consistent serializer prevents churn in git diffs across re-runs.
3. **`--source db` detected via TWO forms**: `--source db` (space) and `--source=db` (equals). Both hit the same exit-1 branch. The user can type either naturally.
4. **`cmdArchive` → `cmdFullRebuild` at end**: Keeps `_index.json` consistent after each archive batch without requiring a separate user invocation. Tradeoff: archive is slightly slower, but invariant is preserved.
5. **`_archived/YYYY-MM-DD/` dir name**: Same-day re-runs merge into existing dir; `mkdirSync(…, { recursive: true })` is idempotent.
6. **Thresholds as module-level constants, not flags**: PRD §5.3 decided the values (150/170/180/365). Making them flags would invite drift. `--older-than-archived=<days>` is the one exception (purge threshold varies by use case).
7. **Audit file `type: audit`**: The schema already accepts this enum value. `ttl: never` because audit snapshots are point-in-time artifacts, not TTL-managed content.
8. **Inline YAML serializer does NOT quote all strings**: Only quotes when `needsQuoting()` returns true (colon+space, trailing colon, leading reserved char, keywords like true/false/null). Match with parser: bare scalars like `deprecated_by: user:antonio` parse correctly (per Plan 149-03 serializer decision).

- **Found during:** Task 2 (first test suite run, Test 9)
- **Issue:** The `writeFixture` helper quotes scalar string values (e.g., `status: "deprecated"`) to be unambiguous for the parser. The `extractFrontmatterField` test helper returned the raw value including the quotes, so `expect(...).toBe('deprecated')` failed with received `"deprecated"`.
