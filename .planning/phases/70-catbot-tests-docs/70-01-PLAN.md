export interface CatBotTool {

    name: string;
    description: string;

  result: unknown;

  <action>
Add 4 new tool definitions to the TOOLS array in catbot-tools.ts and their corresponding executeTool switch cases:

1. **list_catflows** tool definition:
   - name: 'list_catflows'
   - description: 'Lista todos los CatFlows (tareas) con su estado, modo de escucha y programacion'
   - parameters: empty object (no args)
   - Handler: `db.prepare('SELECT id, name, status, listen_mode, execution_mode, schedule_config, created_at FROM tasks ORDER BY updated_at DESC LIMIT 20').all()`
   - Return result with actions: `[{ type: 'navigate', url: '/catflow', label: 'Ver CatFlows →' }]`

2. **execute_catflow** tool definition:
   - name: 'execute_catflow'
   - description: 'Ejecuta un CatFlow por nombre o ID. IMPORTANTE: Confirma con el usuario antes de ejecutar.'
   - parameters: { identifier: { type: 'string', description: 'Nombre o ID del CatFlow a ejecutar' } }, required: ['identifier']
   - Handler: First try db lookup by ID (`SELECT id, name, status FROM tasks WHERE id = ?`), if not found try by name (`SELECT id, name, status FROM tasks WHERE name = ?`), if still not found try LIKE (`WHERE name LIKE ?` with `%identifier%`). If no match return error. Then call `fetch(baseUrl + '/api/tasks/' + task.id + '/execute', { method: 'POST' })`. Return result with task name and status, actions: `[{ type: 'navigate', url: '/catflow', label: 'Ver CatFlows →' }]`

3. **toggle_catflow_listen** tool definition:
   - name: 'toggle_catflow_listen'
   - description: 'Activa o desactiva el modo escucha (listen_mode) de un CatFlow'
   - parameters: { identifier: { type: 'string', description: 'Nombre o ID del CatFlow' }, enable: { type: 'boolean', description: 'true para activar, false para desactivar' } }, required: ['identifier', 'enable']
   - Handler: Same identifier resolution as execute_catflow (ID -> exact name -> LIKE). Then `db.prepare('UPDATE tasks SET listen_mode = ? WHERE id = ?').run(args.enable ? 1 : 0, task.id)`. Return updated listen_mode, actions: navigate to /catflow.

4. **fork_catflow** tool definition:
   - name: 'fork_catflow'
   - description: 'Duplica un CatFlow existente con un nuevo nombre (copia la tarea y todos sus pasos)'
   - parameters: { identifier: { type: 'string', description: 'Nombre o ID del CatFlow a duplicar' }, new_name: { type: 'string', description: 'Nombre para la copia' } }, required: ['identifier', 'new_name']
   - Handler: Same identifier resolution. Then `fetch(baseUrl + '/api/tasks/' + task.id + '/fork', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: args.new_name }) })`. Return fork result, actions: navigate to /catflow.
