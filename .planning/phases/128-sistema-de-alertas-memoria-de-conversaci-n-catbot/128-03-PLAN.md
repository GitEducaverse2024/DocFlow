   ```typescript
   private chatHistories: Map<number, Array<{ role: string; content: string }>> = new Map();
   ```

2. Importar buildConversationWindow de '@/lib/services/catbot-conversation-memory' al inicio del archivo.

3. Modificar handleCatBotMessage (linea ~477):

   ANTES del fetch:
   ```typescript
   // Accumulate user message in chat history
   const history = this.chatHistories.get(chatId) || [];
   history.push({ role: 'user', content: text });
   ```

   En el body del fetch, reemplazar `messages: [{ role: 'user', content: text }]` por:
   ```typescript
   const windowed = await buildConversationWindow(history as any);
   // ...
   messages: windowed,
   ```

   DESPUES del fetch exitoso, cuando se obtiene data.reply:
   ```typescript
   if (data.reply) {
     history.push({ role: 'assistant', content: data.reply });
   }
   // Cap per-chat history to prevent memory leak
   if (history.length > 100) {
     history.splice(0, history.length - 100);
   }
   this.chatHistories.set(chatId, history);
   ```

   Si el fetch falla (response.ok === false o catch), NO anadir assistant reply pero SI mantener el user message en history.

4. Eviccion por inactividad: Crear metodo privado `cleanupStaleChats()`:
   - Recorrer chatHistories entries
   - Si no hay mensajes nuevos en 24h, eliminar el entry
   - Para trackear actividad, anadir propiedad `private chatLastActivity: Map<number, number> = new Map()`
   - Actualizar chatLastActivity.set(chatId, Date.now()) en handleCatBotMessage
   - En cleanupStaleChats: if (Date.now() - lastActivity > 24*60*60*1000) delete both maps
   - Llamar cleanupStaleChats() al inicio de cada handleCatBotMessage (es O(n) con n=chats, negligible)

5. IMPORTANTE: cuando el usuario hace sudo en Telegram (ya manejado por handleSudo), NO limpiar chatHistories[chatId]. El hilo de conversacion se preserva.
