  <action>
1. En db.ts, crear tabla system_alerts con safe migration pattern (CREATE TABLE IF NOT EXISTS):
   ```sql
   CREATE TABLE IF NOT EXISTS system_alerts (
     id TEXT PRIMARY KEY,
     category TEXT NOT NULL,     -- 'knowledge', 'execution', 'integration', 'notification'
     alert_key TEXT NOT NULL,    -- e.g. 'knowledge_gaps_high', 'task_stuck'
     title TEXT NOT NULL,
     message TEXT,
     severity TEXT DEFAULT 'warning',
     details TEXT,               -- JSON con specifics
     acknowledged INTEGER DEFAULT 0,
     acknowledged_at TEXT,
     created_at TEXT DEFAULT (datetime('now'))
   );
   ```

2. Crear alert-service.ts como singleton siguiendo SummaryService pattern:
   - `static start()` / `static stop()` con setInterval 5min
   - Boot delay: 30s (no interferir con startup)
   - `static async tick()` — wrapper que llama cada check en try-catch individual (un check fallando no bloquea los demas)
   - 7 metodos check privados (ver queries SQL en RESEARCH.md seccion "Alert Conditions SQL Queries"):
     a. checkKnowledgeGaps: catbot.db knowledge_gaps WHERE resolved=0 > 20
     b. checkStagingEntries: catbot.db knowledge_learned WHERE validated=0 > 30
     c. checkStuckTasks: docflow.db tasks WHERE status='running' AND updated_at < datetime('now','-1 hour')
     d. checkOrphanedRuns: docflow.db canvas_runs WHERE status IN ('running','pending') AND started_at < datetime('now','-2 hours')
     e. checkFailingConnectors: docflow.db connector_logs WHERE status='error' AND created_at > datetime('now','-1 hour') GROUP BY connector_id HAVING count >= 3
     f. checkStaleSyncs: docflow.db drive_sync_jobs WHERE is_active=1 AND last_synced_at < datetime('now','-'||(sync_interval_minutes*2)||' minutes')
     g. checkUnreadNotifications: docflow.db notifications WHERE read=0 > 50
   - Funcion `insertAlert(category, alert_key, title, message, severity, details)` que hace INSERT solo si no existe alert unacknowledged con misma category+alert_key
   - Funcion `getAlerts(pendingOnly: boolean)` — retorna alertas, si pendingOnly=true solo acknowledged=0
   - Funcion `acknowledgeAll()` — UPDATE system_alerts SET acknowledged=1, acknowledged_at=datetime('now') WHERE acknowledged=0
   - Cleanup: en cada tick, DELETE alertas acknowledged con created_at > 30 dias

   IMPORTANTE: usar `process['env']` (bracket notation). Usar `import { generateId } from '@/lib/utils'` para IDs. Importar db de '@/lib/db' y getCatbotDb de '@/lib/catbot-db'.

3. Crear test file con vitest. Mockear db y catbotDb con in-memory stubs. Testear cada condicion + dedup + getAlerts + acknowledgeAll.
