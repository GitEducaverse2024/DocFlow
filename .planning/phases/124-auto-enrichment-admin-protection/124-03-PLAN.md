    2. `countUserData(userId: string): { profile: boolean; conversations: number; recipes: number; summaries: number; learned: number }`:

       - SELECT COUNT(*) FROM conversation_log WHERE user_id = ?

       - Si 'summaries' en dataTypes: DELETE FROM summaries WHERE user_id = ?

    IMPORTANTE: knowledge_learned NO tiene user_id (son aprendizajes globales de la plataforma). No incluir 'learned' como data_type de deleteUserData. Si el admin quiere borrar learned entries, usa admin_validate_learned con action=reject.

<task type="auto">
  <name>Task 2: Admin sudo tools + safe delete confirmation</name>
  <files>app/src/lib/services/catbot-sudo-tools.ts, app/src/lib/__tests__/catbot-learned.test.ts</files>
  <action>
    1. Anadir 4 tool definitions al array SUDO_TOOLS en catbot-sudo-tools.ts:

       a) `admin_list_profiles`: Lista todos los perfiles de usuario en catbot.db.
          - Params: {} (sin parametros)
          - Handler: llama getAllProfiles(), retorna array de perfiles (masking known_context para brevedad)

       b) `admin_delete_user_data`: Elimina datos de un usuario. Sigue patron safe delete de Holded.
          - Params: user_id (string, required), data_types (array de strings: profile|conversations|recipes|summaries), confirmed (boolean)
          - Handler:
            - Si confirmed !== true: llama countUserData(user_id), retorna CONFIRM_REQUIRED con resumen de lo que se borrara
            - Si confirmed === true: llama deleteUserData(user_id, data_types), retorna success con detalle

       c) `admin_validate_learned`: Valida o rechaza un learned_entry.
          - Params: entry_id (string, required), action (enum: validate|reject, required)
          - Handler: si action=validate, llama adminValidate(entry_id); si action=reject, llama adminReject(entry_id)

       d) `admin_list_learned`: Lista learned_entries con filtros opcionales.
          - Params: validated (boolean, optional), knowledge_path (string, optional)
          - Handler: llama getLearnedEntries({ validated, knowledgePath: knowledge_path }), retorna array

    2. Anadir cases en executeSudoTool switch para los 4 nuevos tools.

    3. Imports necesarios al inicio de catbot-sudo-tools.ts:
       - import { getAllProfiles, countUserData, deleteUserData, getLearnedEntries } from '@/lib/catbot-db'
       - import { adminValidate, adminReject } from '@/lib/services/catbot-learned'

    4. Anadir tests a catbot-learned.test.ts:
       - Test "confirm required": admin_delete_user_data sin confirmed retorna CONFIRM_REQUIRED con conteos
       - Test "confirm deletes": admin_delete_user_data con confirmed=true borra los datos
       - Test "admin validate promotes": admin_validate_learned con action=validate setea validated=1
       - Test "admin reject deletes": admin_validate_learned con action=reject elimina la entry

    IMPORTANTE: Todos estos tools ya estan gated por el sistema sudo existente (solo se ejecutan si sudoActive=true en route.ts). NO necesitan permission gate adicional. El patron safe delete es: primera llamada sin confirmed retorna preview, segunda con confirmed=true ejecuta.
