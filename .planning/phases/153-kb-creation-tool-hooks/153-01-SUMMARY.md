## Decisions Made

See `key-decisions` in frontmatter. Notable:

- **`_sync_failures.md` is the append target, NOT `_audit_stale.md`.** The latter is fully regenerated by `scripts/kb-sync.cjs --audit-stale` (verified in RESEARCH §Major Conflict #1) — any hook entries there would be silently wiped on the next CLI run.
- **`change_log` seeded in the lazy header.** The schema (line 82-95 of `frontmatter.schema.json`) requires `change_log` with `minItems: 1`. The plan's proposed header did not include `change_log`; caught during implementation, corrected in-module.
- **Never-throws is total.** Even `mkdirSync` failures from a bad `KB_ROOT` are swallowed by the outer try/catch (verified by Test 4.2).

## Deviations from Plan

**1. [Rule 1 - Bug fix] Added `change_log` entry to lazy frontmatter header**
- **Found during:** Task 3 implementation.
- **Issue:** The plan's proposed header in `<action>` omits the `change_log` YAML key. `frontmatter.schema.json` requires `change_log: array, minItems: 1`. Without this, a file written by `markStale` would fail schema validation if someone later ran `validate-kb.cjs` with `_sync_failures.md` NOT excluded (belt-and-braces correctness).
- **Fix:** Added `change_log:\n  - { version: 1.0.0, date: ${dateOnly}, author: kb-audit, change: Initial creation by markStale hook }` to the header block between `source_of_truth: null` and `ttl: never`.
- **Files modified:** `app/src/lib/services/kb-audit.ts`
- **Verification:** Test 1 asserts `expect(contents).toMatch(/^change_log:/m)` and passes.
- **Committed in:** `9515585` (Task 3 GREEN).

**2. [Rule 1 - Bug fix] Test 4 split into two cases for total-throws-coverage**
- **Found during:** Task 3 test authoring.
- **Issue:** Plan §behavior Test 4 only covers `fs.appendFileSync` throwing. The implementation also calls `mkdirSync` which can throw if `KB_ROOT`'s parent is a regular file. One test covered only the half.
- **Fix:** Added a second `it(...)` inside the "never throws" describe block that points `KB_ROOT` at a path whose parent is a file, forcing `mkdirSync` to fail.
- **Files modified:** `app/src/lib/__tests__/kb-audit.test.ts`
- **Verification:** Both tests pass. Total 9 tests instead of plan's 8.
- **Committed in:** `467367e` (Task 3 RED).
