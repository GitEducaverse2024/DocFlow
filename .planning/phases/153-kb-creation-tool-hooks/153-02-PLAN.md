<objective>
Hook the 6 real direct-DB-write tool cases in `catbot-tools.ts` to `syncResource()`, so that every successful create/update/delete performed by CatBot through a tool call automatically updates the corresponding `.docflow-kb/resources/**` file and invalidates the Phase 152 index cache. Explicitly leave `update_cat_paw` (L2238) untouched because it is a `fetch`-pass-through to the PATCH API route — hooking it would double-fire or read stale state.

Purpose: This is the first of two parallel write paths (tools + API routes) that Phase 153 has to cover. Closing this surface means CatBot-driven create/update/delete becomes the canonical demo of the end-to-end loop: LLM action → DB write → KB sync → next `list_*` sees `kb_entry` populated. Plan 04's oracle chain exercises exactly this path.

Output: `catbot-tools.ts` edited in 6 places with the wrapper pattern from RESEARCH §Hook Recipe §A. One new test file with ≥9 tests. No refactors. No new imports beyond `syncResource`, `invalidateKbIndex`, `markStale`, and `logger` (if not already imported).
</objective>
