<objective>
Implement the IntentWorker background singleton that re-queues failed intents (LLM-driven retry, not code-driven), register it in instrumentation.ts, and wire the PromptAssembler to inject both the "Protocolo de Intents" (P1) and an "Open Intents" context section (P2) so the retry loop actually closes on the user's next conversation turn.

Purpose: Without this plan, INTENT-04 is a dead table and INTENT-02 has no prompt. Critically, without `buildOpenIntentsContext`, the LLM never sees re-queued intents and the retry loop is a no-op (research Open Question 4).
Output: A 60+ line IntentWorker class, instrumentation registration, 2 new PromptAssembler sections with budget-aware text, and Wave 0 tests covering the state machine.
</objective>
