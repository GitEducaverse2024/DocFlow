<feature>
  <name>MidService: Schema, CRUD, Seeds, Markdown Export</name>
  <files>app/src/lib/services/mid.ts, app/src/lib/services/__tests__/mid.test.ts, app/src/lib/db.ts</files>
  <behavior>
    MID-01 Schema:
    - model_intelligence table has columns: id, model_key (UNIQUE), display_name, provider, tier, best_use, capabilities (JSON), cost_tier, cost_notes, scores (JSON), status, auto_created, created_at, updated_at
    - tier defaults to 'Libre', status defaults to 'active', auto_created defaults to 0

    MID-02 Fields:
    - getAll() returns array of MidEntry objects with all fields
    - getById(id) returns single MidEntry or null
    - create(data) inserts entry with generateId(), returns id
    - update(id, partial) updates only provided fields + updated_at, returns boolean

    MID-03 Seeds:
    - When table has 0 rows, seedModels() inserts ~15-20 models
    - Seeds include: Claude Opus 4, Sonnet 4, Haiku 3.5 (Elite/Pro), GPT-4o, 4o-mini, Gemini 2.5 Pro, 2.5 Flash, Gemma 4 variants (E2B, E4B, 26B, 31B), Llama 3.x, Mistral, Qwen
    - Each seed has tier, best_use, capabilities array, cost_tier, scores object
    - When table has >0 rows, seedModels() does nothing (never overwrites)

    MID-04 Markdown:
    - midToMarkdown() returns string grouped by tier (Elite, Pro, Libre)
    - Each model shows: display_name (provider), best_use, cost, capabilities, scores
    - Retired models excluded
    - Supports compact mode (1-2 lines per model) via optional parameter
    - Output cached with cacheGet/cacheSet (key: 'mid:markdown', TTL: 5min)

    MID-07 Inactive preservation:
    - Models with status 'inactive' appear in getAll() (unless filtered)
    - Models with status 'retired' excluded from markdown export but accessible via getAll({status:'all'})
    - update() can change status without affecting other fields
  </behavior>
  <implementation>
    1. db.ts: Add CREATE TABLE IF NOT EXISTS model_intelligence with all columns per research schema. Add seed block after table creation (COUNT=0 check, INSERT OR IGNORE for ~15-20 models).

    2. mid.ts: Create MidService with:
    - MidEntry type/interface matching table columns (with parsed JSON for capabilities/scores)
    - MidRow type (raw DB row with string JSON fields)
    - getAll(opts?: {status?: string}): MidEntry[] -- defaults to excluding retired
    - getById(id: string): MidEntry | null
    - create(data: CreateMidInput): string -- returns generated id
    - update(id: string, data: Partial<UpdateMidInput>): boolean -- dynamic SET clause for partial updates, JSON.stringify for capabilities/scores fields
    - midToMarkdown(compact?: boolean): string -- tier-grouped, cached 5min
    - syncFromDiscovery(inventory: ModelInventory): {created: number, skipped: number} -- creates basic entries for models not in MID
    - Helper: parseRow(row: MidRow): MidEntry -- safely JSON.parse capabilities/scores with fallback

    3. CRITICAL conventions:
    - Use process['env'] bracket notation (not process.env)
    - Use generateId() from @/lib/utils (not crypto.randomUUID)
    - Use logger from @/lib/logger for seed events and errors
    - Wrap JSON.parse in try-catch with fallback to empty array/object
    - Seeds in Spanish for best_use descriptions (project language)
  </implementation>
</feature>
