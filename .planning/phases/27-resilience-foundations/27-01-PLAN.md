Create three new files in `app/src/lib/`:

**retry.ts (RESIL-01):**
- Export `RetryOptions` interface: `{ maxAttempts?: number (default 3), baseDelayMs?: number (default 500), maxDelayMs?: number (default 10000), shouldRetry?: (error: Error) => boolean }`
- Export `async function withRetry<T>(fn: () => Promise<T>, opts?: RetryOptions): Promise<T>`
- Loop up to `maxAttempts`. On catch: if last attempt or `shouldRetry` returns false, rethrow. Otherwise sleep with exponential backoff: `Math.min(baseDelayMs * 2^(attempt-1), maxDelayMs)` plus jitter `* (0.75 + Math.random() * 0.5)`
- Default `shouldRetry`: return true if error message (lowercased) includes `econnrefused`, `timeout`, `aborted`, `503`, `502`, `504`, or `error.name === 'AbortError'`. Return false for all other errors (4xx, parse errors, etc.)
- Import `logger` from `./logger` and call `logger.warn('Retry attempt', { attempt, maxAttempts, error: lastError.message })` on each retry
- Private `sleep(ms: number)` helper using `new Promise(resolve => setTimeout(resolve, ms))`

**cache.ts (RESIL-03):**
- Module-level `const store = new Map<string, { data: unknown; expiresAt: number }>()`
- Export `function cacheGet<T>(key: string): T | null` — returns `entry.data as T` if entry exists and `Date.now() < entry.expiresAt`, else deletes stale entry and returns null
- Export `function cacheSet<T>(key: string, data: T, ttlMs: number): void` — sets `{ data, expiresAt: Date.now() + ttlMs }`
- Export `function cacheInvalidate(key: string): void` — deletes from store
- Export `function cacheInvalidatePrefix(prefix: string): void` — iterates store keys, deletes any starting with prefix (used for `cacheInvalidatePrefix('dashboard:')`)
- CRITICAL: Only successful responses should be cached. The cache module itself does not enforce this — callers must only call cacheSet on success. Add a JSDoc comment on cacheSet stating this.

**logger.ts (RESIL-04):**
- `import fs from 'fs'` and `import path from 'path'`
- `const LOG_DIR = process['env']['LOG_DIR'] || '/app/data/logs'` (bracket notation)
- On module load: `fs.mkdirSync(LOG_DIR, { recursive: true })` to ensure dir exists, wrapped in try/catch
- On module load: call `rotateLogs()` which reads LOG_DIR, parses `app-YYYY-MM-DD.jsonl` filenames, deletes files older than 7 days. Use `fs.readdirSync` + `fs.unlinkSync`. Wrap in try/catch.
- `function getLogPath(): string` — returns `path.join(LOG_DIR, 'app-' + new Date().toISOString().split('T')[0] + '.jsonl')`
- `function writeLog(level: LogLevel, message: string, data?: Record<string, unknown>): void` — builds JSON line `{ ts: new Date().toISOString(), level, message, ...data }`, appends with `fs.appendFile(getLogPath(), JSON.stringify(entry) + '\n', () => {})` (fire-and-forget callback, NOT sync)
- Export `type LogLevel = 'info' | 'warn' | 'error'`
- Export `const logger = { info: (msg, data?) => writeLog('info', msg, data), warn: (msg, data?) => writeLog('warn', msg, data), error: (msg, data?) => writeLog('error', msg, data) }`
- Do NOT use `fs.appendFileSync` — this would block the event loop
