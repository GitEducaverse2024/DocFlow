Then add a CREATE TABLE for catflow_triggers (use CREATE TABLE IF NOT EXISTS pattern):

```sql
CREATE TABLE IF NOT EXISTS catflow_triggers (
  id TEXT PRIMARY KEY,
  source_task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  source_run_id TEXT,
  source_node_id TEXT,
  target_task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  payload TEXT,
  status TEXT DEFAULT 'pending',
  response TEXT,
  created_at TEXT DEFAULT (datetime('now')),
  completed_at TEXT
);
```

Place the CREATE TABLE statement in the main db.exec block where other tables are created (after the existing table definitions), OR as a separate db.exec call after the ALTER TABLE migrations. The existing pattern uses a single large db.exec for initial tables and separate try/catch for ALTER TABLE migrations -- follow the same pattern by adding the CREATE TABLE IF NOT EXISTS in a separate db.exec call near the ALTER TABLE block.
