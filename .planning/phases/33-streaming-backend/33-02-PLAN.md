   ```
   (async () => {
     try {
       send('start', { timestamp: Date.now() });
       let totalInputTokens = 0;
       let totalOutputTokens = 0;

       const allActions = [];

       for (let iteration = 0; iteration < maxIterations; iteration++) {

         // Use streamLiteLLM for this iteration
         await streamLiteLLM(
           { model, messages: llmMessages, max_tokens: 2048, tools: tools.length > 0 ? tools : undefined },
           {
             onToken: (token) => {
               iterationContent += token;
               send('token', { content: token });
             },
             onToolCall: (tc) => {
               // Tool call detected — will be processed after stream completes
               // The onToolCall fires when finish_reason === 'tool_calls'
             },
             onDone: (usage) => {
               totalInputTokens += usage?.prompt_tokens || 0;
               totalOutputTokens += usage?.completion_tokens || 0;
             },
             onError: (error) => { throw error; },
           }
         );

         // After streaming completes, check if the model produced tool calls
         // We need to detect this from the streamed content — streamLiteLLM accumulates tool calls
         // and fires onToolCall. We need to collect them.

         // REVISED APPROACH: Collect tool calls during streaming
         const pendingToolCalls = [];
         // ... (see detailed approach below)
       }
     } catch (error) { ... }
   })();
   ```

   **DETAILED streaming tool-call loop approach:**

   For each iteration (0 to maxIterations-1):
   a. Collect tool calls in a local array via onToolCall callback
   b. Stream tokens via onToken -> send('token', ...)
   c. After `streamLiteLLM` resolves:
      - If no tool calls were collected -> this is the final text response, break the loop
      - If tool calls were collected:
        - Push an assistant message to `llmMessages` with `content: iterationContent` and `tool_calls` array
        - For each tool call:
          - Emit `send('tool_call_start', { name: tc.function.name, id: tc.id })`
          - Execute the tool (regular or sudo, same logic as current non-streaming code — check `isSudoTool`, check `sudoActive`, call `executeTool` or `executeSudoTool`)
          - Emit `send('tool_call_result', { id: tc.id, name: tc.function.name, result: toolResult.result })`
          - Push tool result message to `llmMessages`
          - Collect in `allToolResults` and `allActions`
        - Continue to next iteration (next LLM call with tool results)

   d. After loop ends (either break on final text or max iterations):
      - Call `logUsage(...)` with accumulated totals
      - Emit `send('done', { usage: { input: totalInputTokens, output: totalOutputTokens }, tool_calls: allToolResults, actions: allActions, sudo_required: sudoRequired, sudo_active: !!sudoActive })`
      - Call `close()`

   e. Wrap entire loop in try/catch -> `send('error', { message })` + `close()`
