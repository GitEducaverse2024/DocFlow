**Fix A: Reporting protocol (catbot-prompt-assembler.ts)**

Modificar `buildReportingProtocol()` (linea 802) para cambiar la regla contradictoria. La regla actual dice "NO reportes paso a paso durante la ejecucion. Ejecuta TODO y reporta al final." pero el requisito EVAL-02 exige reporting paso a paso durante canvas construction.

Reemplazar la funcion completa con:

```typescript
function buildReportingProtocol(): string {
  return `## Protocolo de Reporting (OBLIGATORIO)

Cuando construyas o modifiques un canvas (multiples tool calls), reporta el progreso al usuario:

### Formato
- Cada paso exitoso: "✓ [Accion]: [resultado breve]"
  Ejemplo: "✓ canvas_create: Canvas 'Email Classifier' creado"
  Ejemplo: "✓ canvas_add_node: Normalizador JSON agregado (agent, model: canvas-classifier)"
  Ejemplo: "✓ canvas_add_edge: Normalizador → Clasificador conectado"

- Cada paso fallido: "✗ [Accion]: [error] — [solucion propuesta]"
  Ejemplo: "✗ canvas_add_edge: OUTPUT es terminal — conectar al nodo anterior en su lugar"

### Reglas
- Reporta CADA tool call de canvas con ✓ o ✗ inmediatamente despues de ejecutarla.
- Si un paso falla, PARA inmediatamente. Reporta lo completado con ✓ y el fallo con ✗.
- Propone una solucion o revision para el fallo.
- Solo texto legible para el usuario — nada de JSON tecnico ni dumps de datos.
- Consulta CatBrain DoCatFlow para errores y soluciones conocidas antes de proponer una solucion nueva.
- Al finalizar, incluye un resumen: "Resumen: N nodos creados, M edges conectados, canvas listo."`;
}
```

Los cambios clave:
1. ELIMINADA la regla "NO reportes paso a paso" — ahora dice "Reporta CADA tool call"
2. Cambiados los ejemplos para usar tool names reales (canvas_add_node, canvas_add_edge)
3. Agregado resumen al final
4. Mantiene la regla de parar en fallo

// Phase 144: Enrich Skill Orquestador with label fidelity rule
const ORQUESTADOR_PART_18 = `

## PARTE 18 — FIDELIDAD DE LABELS Y NOMBRES

REGLA ABSOLUTA: Cuando el usuario especifica nombres/labels exactos para nodos, CatBot DEBE usar esos EXACTOS nombres. No inventar nombres alternativos, no "mejorar" los nombres, no traducir.

Ejemplos:
- Usuario: "anade un nodo CONDITION llamado 'Tiene producto?'" → label DEBE ser "Tiene producto?" (NO "Evaluacion de Calidad", NO "Filtro")
- Usuario: "crea un OUTPUT 'Descartado'" → label DEBE ser "Descartado" (NO "Salida Final", NO "Fin")
- Usuario: "nodo AGENT 'Clasificador de Producto'" → label DEBE ser "Clasificador de Producto"

Si el usuario NO especifica un nombre, CatBot puede elegir un nombre descriptivo apropiado.
Si el usuario especifica un nombre vago (ej: "un nodo agente"), CatBot elige un nombre descriptivo.
Si el usuario especifica un nombre concreto (ej: "llamado 'Filtro Spam'"), CatBot usa ese nombre EXACTO.`;

try {
  const orqSkill2 = db.prepare("SELECT id, instructions FROM skills WHERE name = 'Orquestador CatFlow' LIMIT 1").get() as { id: string; instructions: string } | undefined;
  if (orqSkill2 && !orqSkill2.instructions.includes('FIDELIDAD DE LABELS')) {
    db.prepare("UPDATE skills SET instructions = ?, updated_at = ? WHERE id = ?").run(

      new Date().toISOString(),
      orqSkill2.id
    );
    logger.info('system', 'Phase 144: Skill Orquestador enriched with label fidelity rule');
  }
} catch (e) { logger.error('system', 'Phase 144 skill label update error', { error: (e as Error).message }); }
