Ejecutar cada uno de los 10 tests de la scorecard original contra CatBot usando curl al endpoint POST http://localhost:3500/api/catbot/chat. El endpoint es SSE streaming — parsear las respuestas para extraer el contenido de texto y los tool_calls.

Para cada test, crear una conversacion nueva (conversation_id unico). Los 10 prompts son:

1. **Tipos de nodos**: "Que tipos de nodos puedo usar en un Canvas? Lista todos los disponibles con descripcion."
2. **Busqueda de recursos (CatPaws)**: "Que CatPaws pueden enviar emails?" — Verificar que CatBot usa list_cat_paws tool, NO responde de memoria.
3. **Config completa (plantillas)**: "Que plantillas de email tengo y cuales estan vacias? Listamelas todas." — Verificar que usa list_email_templates.
4. **Planificacion**: "Disena un CatFlow para procesar emails entrantes y responder con la plantilla correcta segun el producto" — Solo planificacion, no ejecucion.
5. **Crear canvas**: "Crea un canvas llamado 'Eval Test v28' en modo mixed" — Verificar que crea correctamente con START.
6. **Anadir nodo + edge**: "En el canvas 'Eval Test v28', anade un nodo AGENT llamado 'Clasificador de Producto' con instrucciones: 'INPUT: {emails: [{from, subject, body}]}\nOUTPUT: {emails: [{from, subject, body, producto, template_id}]}\nClasifica cada email por producto Educa360. Mapeo: K12->bc03e496, Simulator->d7cc4227, REVI->9f97f705, Educaverse->155c955e. Si no reconoces el producto, template_id=null.' Conecta START al Clasificador." — Verificar que instrucciones persisten (usar canvas_get despues).
7. **Multiples nodos + edges**: "En el canvas 'Eval Test v28', anade: 1) Un nodo CONDITION llamado 'Tiene producto?' con instrucciones 'Evalua si template_id es distinto de null', 2) Un nodo OUTPUT llamado 'Descartado' para emails sin producto, 3) Un nodo OUTPUT llamado 'Procesado' para emails con producto. Conecta Clasificador->Condition, Condition[no]->Descartado, Condition[yes]->Procesado." — Verificar que NO escala a async, construye sincronamente.
8. **Verificacion canvas**: "Muestrame el canvas 'Eval Test v28' completo con todos los nodos y conexiones" — Verificar que reporta correctamente todos los nodos, edges, sourceHandle.
9. **Recuperacion de errores**: "En el canvas 'Eval Test v28', conecta un nodo AGENT despues del nodo OUTPUT 'Descartado'" — Verificar que CatBot RECHAZA la operacion explicando que OUTPUT es terminal.
10. **Reporting paso a paso**: "Crea un canvas nuevo 'Eval Reporting Test' en modo mixed y construye un mini-flujo: START -> Agent 'Lector' -> Condition '?Hay datos?' -> OUTPUT 'Vacio' y OUTPUT 'Listo'. Reporta despues de cada accion." — Verificar que reporta con check/cross despues de cada tool call.

Para cada test, enviar el prompt via curl:
```bash

  -d '{"message":"PROMPT","conversation_id":"eval-144-testNN","model":"gemini-main"}' \
  2>&1 | timeout 120 cat
```

- Test 7 (Multiples): 10 si construye sincronamente con sourceHandle yes/no. 7 si construye pero escala a async. 3 si falla.
- Test 8 (Verificacion): 10 si reporta nodos+edges+sourceHandle correctamente. 7 si parcial. 3 si incorrecto.
- Test 9 (Recuperacion): 10 si rechaza la operacion con mensaje claro. 5 si advierte pero ejecuta. 0 si ejecuta sin advertir.
