# Plan 62-01: Status API + Types + Canvas Step UI + Cycle Progress

## Goal
Extend the status API to return canvas execution progress and cycle data, update TypeScript types for all v15 step types, render canvas step progress in the task detail page, and display cycle progress for variable mode tasks.

## Context
- Status API (`app/src/app/api/tasks/[id]/status/route.ts`) currently returns step-level data but no canvas_run progress or cycle metadata
- Task detail page has `TaskStepDetail.type` limited to `'agent' | 'checkpoint' | 'merge'` — missing `canvas`, `fork`, `join`
- `getStepIcon()` only handles `checkpoint` and `merge`, defaults to `Bot`
- Canvas runs store `node_states` JSON, `current_node_id`, and `execution_order` in the `canvas_runs` table
- Task has `execution_mode`, `execution_count`, `run_count` columns already populated by Phase 60
- `canvas_runs.metadata` contains `parent_task_id` and `parent_step_id` linking back to the task

## Tasks

### Task 1: Extend status API with canvas_run progress and cycle data

**File**: `app/src/app/api/tasks/[id]/status/route.ts`

1. After fetching steps, find any step with `type='canvas'` AND `status='running'`
2. For each running canvas step, query `canvas_runs` via the step's `canvas_id`:
   ```sql
   SELECT cr.id, cr.status, cr.node_states, cr.current_node_id, cr.execution_order,
          c.name as canvas_name
   FROM canvas_runs cr
   JOIN canvases c ON c.id = cr.canvas_id
   WHERE cr.metadata LIKE '%"parent_step_id":"' || step_id || '"%'
     AND cr.status = 'running'
   ORDER BY cr.created_at DESC LIMIT 1
   ```
3. Parse `node_states` JSON, count total nodes and completed nodes, get current node name from canvas flow_data
4. Attach canvas progress to the step in the response:
   ```typescript
   canvas_progress?: {
     canvas_run_id: string;
     canvas_name: string;
     total_nodes: number;
     completed_nodes: number;
     current_node_name: string | null;
   }
   ```
5. Add `run_count` and `execution_count` to the top-level status response by selecting from tasks table:
   ```sql
   SELECT id, status, started_at, total_tokens, total_duration,
          execution_mode, execution_count, run_count
   FROM tasks WHERE id = ?
   ```
6. Also include `fork_group`, `branch_index`, `branch_label` in the step SELECT query for fork display (used in Plan 62-03)

### Task 4: Canvas step progress card

**File**: `app/src/app/tasks/[id]/page.tsx`

Inside the step rendering loop (after step header), when `step.type === 'canvas'`:

1. Find the matching status step to get `canvas_progress` data
2. If canvas_progress exists and step is running, render:
   ```tsx
   <div className="px-4 pb-3 space-y-2">
     <div className="flex items-center gap-2 text-sm text-zinc-400">
       <Workflow className="w-4 h-4 text-violet-400" />
       <span className="text-zinc-200 font-medium">{canvas_progress.canvas_name}</span>
       <span className="text-zinc-500">·</span>
       <span>{t('detail.canvasNode', { current: canvas_progress.completed_nodes + 1, total: canvas_progress.total_nodes })}</span>
     </div>
     {/* Progress bar */}
     <div className="h-1.5 bg-zinc-800 rounded-full overflow-hidden">
       <div
         className="h-full bg-violet-500 rounded-full transition-all duration-500"
         style={{ width: `${(canvas_progress.completed_nodes / canvas_progress.total_nodes) * 100}%` }}
       />
     </div>
     {canvas_progress.current_node_name && (
       <div className="text-xs text-zinc-500">
         {t('detail.canvasCurrentNode')}: <span className="text-zinc-400">{canvas_progress.current_node_name}</span>
       </div>
     )}
     {/* "Ver canvas en tiempo real" link — modal handled in Plan 62-02 */}
     <button
       className="text-xs text-violet-400 hover:text-violet-300 underline underline-offset-2"
       data-canvas-run-id={canvas_progress.canvas_run_id}
       id={`canvas-live-${step.id}`}
     >
       {t('detail.canvasLiveView')}
     </button>
   </div>
   ```

### Task 5: Cycle progress display for variable mode

**File**: `app/src/app/tasks/[id]/page.tsx`

1. Track `runCount` and `executionCount` from status polling (update from StatusResponse)
2. When `task.execution_mode === 'variable'` and task is running, render above the pipeline:
   ```tsx
   <div className="mb-4 bg-zinc-900 border border-zinc-800 rounded-lg p-4">
     <div className="flex items-center justify-between mb-2">
       <span className="text-sm font-medium text-zinc-200">
         {t('detail.cycleLabel', { current: runCount + 1, total: executionCount })}
       </span>
       <span className="text-xs text-zinc-500">
         {t('detail.cycleStepInfo', { step: currentStepIndex + 1, totalSteps: task.steps.length })}
         {' · '}
         {Math.round(overallProgress)}%
         {' · '}
         {formatTime(elapsedTime)}
         {task.total_tokens > 0 && ` · ${formatTokens(task.total_tokens)} tokens`}
       </span>
     </div>
     <div className="h-2 bg-zinc-800 rounded-full overflow-hidden">
       <div
         className="h-full bg-gradient-to-r from-violet-500 to-purple-600 rounded-full transition-all duration-500"
         style={{ width: `${overallProgress}%` }}
       />
     </div>
   </div>
   ```
3. Calculate `overallProgress`:
   ```typescript
   const cycleProgress = executionCount > 0 ? (runCount / executionCount) : 0;
   const stepProgress = task.steps.length > 0 ? (completedSteps / task.steps.length) : 0;
   const overallProgress = ((cycleProgress + stepProgress / executionCount) * 100);
   ```
