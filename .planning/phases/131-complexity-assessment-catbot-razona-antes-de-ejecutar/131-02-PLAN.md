<objective>
Wire the prompt protocol created in Plan 01 into runtime behavior: parse the `[COMPLEXITY:...]` prefix emitted by the LLM, persist every decision, and block the tool loop when classification is `complex`. Extend `queue_intent_job` to accept a free-form `description` so the Phase 130 strategist can handle multi-step requests without a specific tool anchor.

Purpose: close the real-world timeout bug (Holded Q1 case). After this plan, `/api/catbot/chat` never executes a 60s+ multi-tool chain without first asking the user. Every classification — right or wrong — lands in `complexity_decisions` for auditability.

Output: new parser module + gate integrated in both streaming and non-streaming paths of route.ts (ONCE per turn, iteration 0 only) + queue_intent_job accepting description + IntentJobExecutor handling the `__description__` synthetic tool_name.
</objective>
