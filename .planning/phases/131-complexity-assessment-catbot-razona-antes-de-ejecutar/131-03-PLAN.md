<objective>
Add two safety nets on top of the Plan 02 gate:

1. Self-check escalation — if CatBot misclassified a complex request as simple and keeps emitting tool_calls past iteration 3, route.ts breaks the loop, queues the remaining work as a CatFlow via createIntentJob, and tells the user.

2. 60s progress reporting — IntentJobExecutor notifies the original channel (Telegram or web) at most once per 60s (or immediately on pipeline_phase change), so the user sees the async pipeline is alive.

Purpose: classification is probabilistic — the self-check catches false-negatives at runtime. Progress reporting closes the UX loop started by the gate: "recibirás reportes cada 60 segundos" becomes a real observable behavior.

Output: self-check branch in route.ts tool loop + extended IntentJobExecutor.notifyProgress with Map-based throttling + new `pipeline_progress` NotificationType + IntentJobExecutor test coverage.
</objective>
