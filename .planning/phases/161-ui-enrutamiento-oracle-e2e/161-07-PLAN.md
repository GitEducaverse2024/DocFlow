<objective>
Close **Gap A** from 161-VERIFICATION.md: the model-resolution chain at `app/src/app/api/catbot/chat/route.ts:121` reads the legacy `catbot_config.model` settings row BEFORE the v30.0 alias config (`cfg.model`). In production this lets a stale `catbot_config.model='gemini-main'` value silently shadow the alias set via `set_catbot_llm({model: "claude-opus"})`. The request then hits Gemini 3 (not Opus as the user requested), LiteLLM auto-translates `reasoning_effort` to `thinking_level`, collides with our `thinking` object, and returns HTTP 400. The user sees the `set_catbot_llm` tool respond `success:true` but the next message fails ŌĆö TOOL-03 contract silently broken.

Purpose: Restore the v30.0 invariant "alias is the source of truth for CatBot's model" (locked in Phase 159-01 CONTEXT) and unblock VER-03 evidence capture. Without this fix, even a correct Plan 161-03 logger cannot fire because the request never reaches a reasoning-capable model.

Output: One-line priority inversion at `route.ts:121` (`requestedModel || cfg.model || catbotConfig.model`) plus a regression test that locks the contract (both rows present in DB, alias wins). Docker rebuild verified via smoke curl.

**Decision ŌĆö Option 1 (priority inversion) chosen** over Option 2 (atomic dual-write in `set_catbot_llm`) and Option 3 (remove `catbot_config.model` entirely):

| Option | Blast radius | Risk | Verdict |
|--------|--------------|------|---------|
| 1. Invert priority (THIS PLAN) | 1 line, 1 file | Existing callers that intentionally set `catbot_config.model` via `/settings` UI fall to back-compat path ŌĆö same behavior as a fresh DB where that row is absent. `cfg.model` is always non-null (resolveAliasConfig seeds defaults). | **Chosen** ŌĆö surgical, respects v30.0 "alias is source of truth" contract. |
| 2. Dual-write in `set_catbot_llm` | 2 files (tool + handler) | Atomicity risk; diverges two writable sources; still leaves `/settings` UI as an orthogonal mutation point. | Rejected ŌĆö treats symptom, not cause. |
| 3. Remove `catbot_config.model` | Ōēź4 files (settings page, API, DB read, tool) | Breaks the Settings UI model dropdown that users rely on when sudo is not active. | Rejected ŌĆö too invasive for a gap-closure plan. |

Side effect locked: `catbotConfig.model` becomes effectively dead code because `cfg.model` is always non-null ŌĆö but the fallback is left in place for defense-in-depth (future refactor of `resolveAliasConfig` could theoretically return undefined; keeping the legacy fallback costs nothing). Dead-code cleanup deferred to v30.1.
