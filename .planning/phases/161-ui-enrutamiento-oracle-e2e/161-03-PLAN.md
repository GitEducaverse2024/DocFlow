<objective>
Add a silent server-side logger that records `reasoning_tokens` from LiteLLM responses in `/api/catbot/chat` (both streaming + non-streaming paths). This is the VER-03 evidence mechanism: when the oracle UAT fires "next message after set_catbot_llm to Opus+high" the log file will contain a JSONL entry `{source: "catbot-chat", message: "reasoning_usage", reasoning_tokens: >0, model, alias: "catbot"}` as proof the passthrough is operational end-to-end.

Purpose: VER-03 requires "siguiente request de CatBot usa reasoning (response incluye `reasoning_content` no-null + metric `reasoning_tokens>0`)". Per CONTEXT.md § Oracle 3/3 mechanism, we use the log as the captured metric (reasoning_content rendering is explicitly deferred to v30.1/FUT-03). Logger is silent for non-reasoning responses so non-reasoning models produce zero log noise.

Output: Extend `StreamCallbacks.onDone` to accept a richer usage shape with `completion_tokens_details.reasoning_tokens`; add a single `logger.info('catbot-chat', 'reasoning_usage', {...})` call in both paths, gated on `reasoning_tokens > 0`; tests asserting the log call fires with correct source/message/payload when reasoning_tokens present, and does NOT fire when absent. Type-level contract guard with `expectTypeOf` ensures signature extension doesn't silently regress.
</objective>
