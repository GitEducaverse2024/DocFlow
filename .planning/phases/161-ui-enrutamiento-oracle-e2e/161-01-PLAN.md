<objective>
Seed LiteLLM shortcut ids (`claude-opus`, `claude-sonnet`, `gemini-main`, `gemma-local`) into `model_intelligence` with capabilities copied from their FQN equivalents, so that `/api/models` + `/api/aliases` + the CatBot `list_llm_models` tool report non-null capabilities in production. This unblocks VER-01 oracle ("¿qué modelos soporto y cuáles piensan?") and the UI conditional rendering (UI-01/02/03) which both depend on non-null `capabilities` from the DB layer.

Purpose: Closes the STATE.md-documented namespace mismatch between LiteLLM gateway shortcuts and `model_intelligence.model_key` FQNs tactically (resolver layer deferred to v30.1). Without this, every model in production returns `capabilities=null` and the oracle + UI conditional controls have nothing to render.

Output: 4 new rows in `model_intelligence` (via `INSERT OR IGNORE`), each with correct `provider`, `tier`, `is_local`, `supports_reasoning`, `max_tokens_cap` values, written idempotently on every bootstrap. Test coverage asserts all 4 shortcut rows land with the expected capability values.
</objective>
