**Seccion 2: CatFlow Email Classifier Pilot** (PILOT-02)

Crear canvas con 8 nodos + 9 edges (incluyendo branch spam->OUTPUT):

Nodos (usar UUIDs generados con crypto.randomUUID()):
1. **START** — `initialInput` con 3 emails JSON representativos:
   - Email 1: Lead caliente pidiendo demo de K12 (from: persona real, tipo A)
   - Email 2: Consulta sobre REVI para universidad (from: persona real, tipo B)
   - Email 3: Newsletter/spam de marketing (from: sistema, tipo G)

2. **Normalizador** (type: agent) — model: `canvas-formatter`, instructions con data contract:
   ```
   Extrae de cada email un JSON con exactamente estos 6 campos:
   { "from": "remitente", "subject": "asunto", "date": "fecha", "body_plain": "texto plano primeros 500 chars", "has_attachments": false, "message_id": "id unico" }
   Si recibes multiples emails, devuelve un array JSON. Cada email es un objeto independiente.
   ```
   agentId: buscar en cat_paws un agente tipo normalizador/lector, o usar uno generico. NO inventar UUID.

3. **Clasificador** (type: agent) — model: `canvas-classifier`, instructions:
   ```
   Clasifica cada email. Input: JSON del normalizador. Output por email:
   { ...campos_originales, "producto": "K12|Simulator|REVI|Educaverse|otro|spam", "template_id": "UUID de plantilla Pro-*", "reply_mode": "REPLY_HILO|EMAIL_NUEVO", "confianza": 0.9, "razon": "..." }
   Mapeo producto->template_id: K12->bc03e496, Simulator->d7cc4227, REVI->9f97f705, Educaverse->155c955e, otro->bc03e496 (default K12), spam->null
   Si from contiene blastfunnels/typeform/hubspot -> reply_mode=EMAIL_NUEVO, extraer reply_to_email del body
   Si from es persona real -> reply_mode=REPLY_HILO, reply_to_email=from
   ```

4. **Condition** (type: condition) — `condition`: "Si producto es 'spam', responde NO. Para cualquier otro producto, responde SI."
   sourceHandle yes -> RAG, sourceHandle no -> OUTPUT

5. **RAG** (type: catbrain) — Query al CatBrain DoCatFlow con `{producto} + {subject}`. Buscar catbrain_id real en la BD con `SELECT id FROM catbrains WHERE name LIKE '%DoCatFlow%'`. Si no existe, documentar que hay que crearlo manualmente.

6. **Respondedor** (type: agent) — model: `canvas-writer`, instructions:
   ```
   Genera email de respuesta comercial. Input: contexto RAG + datos del clasificador.
   Usa la plantilla Pro-* correspondiente al template_id. Estructura: saludo personalizado, propuesta de valor del producto con contexto RAG, CTA (agendar demo/llamada), despedida.
   Output: { ...campos_originales, "cuerpo_respuesta": "texto del email", "asunto_respuesta": "Re: {subject}" }
   Maximo 200 palabras. Tono profesional y cercano.
   ```

7. **Gmail** (type: connector) — Conector Gmail existente. Buscar connector_id real con `SELECT id FROM connectors WHERE type = 'gmail'`. Instructions:
   ```
   Envia el email usando el conector Gmail. Si reply_mode=REPLY_HILO, usa reply con message_id. Si reply_mode=EMAIL_NUEVO, usa send a reply_to_email.
   ```

8. **OUTPUT** (type: output) — label: "Pipeline Completado", recibe resultado final + ruta spam

Edges (9 total):
- start -> normalizador
- normalizador -> clasificador
- clasificador -> condition
- condition (yes) -> rag
- condition (no) -> output (ruta spam)
- rag -> respondedor
- respondedor -> gmail
- gmail -> output

Positions: distribuir horizontalmente con 280px entre nodos, condition branches verticalmente.
