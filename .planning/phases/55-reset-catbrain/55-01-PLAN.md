<objective>
Implement the CatBrain reset feature: a POST API endpoint that wipes data while preserving config, and a 2-step confirmation dialog (step 1: show what will be deleted; step 2: type CatBrain name to confirm). Wire the entry modal "Resetear" button to open this dialog instead of navigating.

Purpose: Users need a safe way to wipe a CatBrain's data (sources, processing runs, RAG vectors, files) and start fresh without losing their configuration. This is a destructive operation requiring strong confirmation gates.
Output: Working reset flow triggered from entry modal, with API endpoint, confirmation UI, and i18n support.
</objective>

   DialogContent: Add `onInteractOutside={(e) => { if (resetting) e.preventDefault(); }}` to prevent clicking outside during reset (Pitfall 3 from research).

   Step 1 UI:
   - Title with AlertTriangle icon: `t('reset.title')` using `useTranslations('catbrains')`
   - Description text: if `stats.vectors_count != null && stats.vectors_count > 0`, show `t('reset.step1Description', { sourceCount: stats.sources_count, vectorCount: stats.vectors_count })`. Otherwise show `t('reset.step1NoVectors', { sourceCount: stats.sources_count })`
   - Two buttons: Cancel (outline, calls handleClose(false)) and Continue (red styled like DeleteProjectDialog, calls setStep(2))

   Step 2 UI:
   - Prompt text: `t('reset.step2Prompt', { name: catbrain.name })` with catbrain name in bold `<span className="font-semibold text-zinc-200">`
   - Input field for typing name, same styling as DeleteProjectDialog
   - normalize function: `(s: string) => s.trim().replace(/\s+/g, ' ').normalize('NFC')` — same as DeleteProjectDialog
   - nameMatches: `normalize(confirmText) === normalize(catbrain.name)`
   - Two buttons: Cancel (outline) and Confirm (red bg, DISABLED when `!nameMatches || resetting`)
   - Confirm button shows Loader2 spinner when resetting, text switches between `t('reset.confirm')` and `t('reset.resetting')`

   handleReset (async):
   - Set `resetting = true`
   - `const res = await fetch(\`/api/catbrains/${catbrain.id}/reset\`, { method: 'POST' })`
   - If res.ok: call `onResetComplete()` (parent handles navigation and toast)
   - If not ok: show error toast using sonner `toast.error(t('reset.error'))` — import toast from sonner
   - Finally: set `resetting = false`

   Use same Tailwind classes as DeleteProjectDialog: `max-w-md bg-zinc-950 border-zinc-800` on DialogContent.
