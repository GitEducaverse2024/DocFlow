### Task 3: Create task_schedules and task_bundles tables

Create 2 new tables using the existing CREATE TABLE IF NOT EXISTS pattern.

**File:** `app/src/lib/db.ts`

Add after the `task_templates` CREATE TABLE block (after line 787):

```typescript
// v15.0 — Tasks Unified: schedules + export bundles
db.exec(`
  CREATE TABLE IF NOT EXISTS task_schedules (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    next_run_at TEXT,
    is_active INTEGER DEFAULT 1,
    run_count INTEGER DEFAULT 0,
    last_run_at TEXT,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
  );

  CREATE TABLE IF NOT EXISTS task_bundles (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    bundle_name TEXT NOT NULL,
    bundle_path TEXT NOT NULL,
    manifest TEXT,
    created_at TEXT DEFAULT (datetime('now'))
  );
`);
```

**Table details:**
- **task_schedules** (DATA-06): Stores active schedule state per task. `is_active` toggles the scheduler. `run_count` tracks scheduled executions. `next_run_at` is calculated by the scheduler (Phase 60).
- **task_bundles** (DATA-07): Stores export bundle metadata. `manifest` is the JSON string of manifest.json contents. `bundle_path` is absolute path to ZIP. CASCADE on task deletion.
