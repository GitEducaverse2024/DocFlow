    - Description: "Recomienda el modelo optimo para una tarea basandose en MID. Analiza tipo de tarea, complejidad y presupuesto para sugerir el mejor modelo."
    - Parameters:
      - task_description: { type: 'string', description: 'Descripcion de la tarea o tipo de trabajo' } (required)
      - complexity: { type: 'string', enum: ['low', 'medium', 'high'], description: 'Complejidad estimada de la tarea' }
      - prefer_local: { type: 'boolean', description: 'Preferir modelos locales (Ollama) sobre API' }
    - Implementation:
      1. Get all active MID entries and current inventory
      2. Filter to only available models (cross-reference with inventory)
      3. Build recommendation logic:
         - complexity=low -> prefer Libre tier, warn against Elite
         - complexity=medium -> prefer Pro tier
         - complexity=high -> allow Elite tier
         - If prefer_local, boost Ollama provider models
      4. Return: { recommended: { model_key, display_name, tier, reason }, alternatives: [...top 2 alternatives], warning?: string }
      - warning if recommending Elite for low complexity: "Modelo Elite para tarea simple -- considera Pro o Libre para ahorrar costes"

    - Description: "Cambia el modelo asignado a un alias de routing. IMPORTANTE: Siempre confirma con el usuario antes de ejecutar este cambio."
