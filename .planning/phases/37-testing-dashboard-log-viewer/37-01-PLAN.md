  <action>
    Create 6 API routes. All MUST export `dynamic = 'force-dynamic'`. All MUST use `process['env']` bracket notation.

    **POST /api/testing/run** (app/src/app/api/testing/run/route.ts):
    - Accept optional JSON body `{ section?: string }`. If section provided, map to spec file path (e.g. "projects" -> "e2e/specs/projects.spec.ts").
    - Maintain module-level `currentRun` variable: `{ id: string; status: 'running' | 'passed' | 'failed'; output: string; startedAt: number } | null`.
    - If `currentRun` is not null and status is 'running', return 409 `{ error: 'Ya hay una ejecucion en curso' }`.
    - Generate run ID with `crypto.randomUUID()` — this runs in Node.js context (API route), not browser.
    - Spawn `npx playwright test` with args. If section, add spec file path. Always add `--reporter=list,./e2e/reporters/sqlite-reporter.ts`.
    - Set `cwd` to `process.cwd()` (resolves to /app in Docker).
    - Pass `env: { ...process['env'] }` to spawn.
    - Capture stdout/stderr into currentRun.output. On 'close' event, set status to 'passed' (code 0) or 'failed' (code !== 0), then set `currentRun = null` after 30 seconds (allow status to be polled).
    - Return 200 `{ id, status: 'running' }` immediately.
    - Section-to-file mapping as const object: `{ navigation: 'e2e/specs/navigation.spec.ts', projects: 'e2e/specs/projects.spec.ts', sources: 'e2e/specs/sources.spec.ts', processing: 'e2e/specs/processing.spec.ts', rag: 'e2e/specs/rag.spec.ts', chat: 'e2e/specs/chat.spec.ts', agents: 'e2e/specs/agents.spec.ts', workers: 'e2e/specs/workers.spec.ts', skills: 'e2e/specs/skills.spec.ts', tasks: 'e2e/specs/tasks.spec.ts', canvas: 'e2e/specs/canvas.spec.ts', connectors: 'e2e/specs/connectors.spec.ts', catbot: 'e2e/specs/catbot.spec.ts', dashboard: 'e2e/specs/dashboard.spec.ts', settings: 'e2e/specs/settings.spec.ts' }`.

    **GET /api/testing/status** (app/src/app/api/testing/status/route.ts):
    - Import `currentRun` from the run route module (or use a shared module-level state file). Actually, since Next.js API routes may run in separate module contexts, create a shared state file `app/src/lib/testing-state.ts` that exports `{ currentRun, setCurrentRun }` using a module-level variable. Both run and status routes import from it.
    - Return `{ status: currentRun?.status || 'idle', id: currentRun?.id, output: currentRun?.output }`.

    **GET /api/testing/results** (app/src/app/api/testing/results/route.ts):
    - Accept query param `limit` (default 10).
    - Query: `SELECT * FROM test_runs ORDER BY created_at DESC LIMIT ?`.
    - Return JSON array of test runs. Parse `results_json` from string to array for each row.

    **POST /api/testing/generate** (app/src/app/api/testing/generate/route.ts):
    - Accept JSON body `{ section: string }`.
    - Read the corresponding spec file and route handler source using fs.readFileSync.
    - Section-to-source mapping: route handler at `app/src/app/api/{section}/route.ts` or similar (best effort, read if exists).
    - Call `chatCompletion` from `@/lib/services/llm` with model "gemini-main", provider "litellm", and a system prompt that instructs to generate a Playwright E2E spec in Spanish using the POM pattern.
    - Return `{ code: string }` with the generated test code.

    **GET /api/system/logs** (app/src/app/api/system/logs/route.ts):
    - Accept query params: `level` (info|warn|error), `source` (LogSource values), `search` (text), `limit` (default 200), `date` (YYYY-MM-DD, default today).
    - Read log file at `${LOG_DIR}/app-${date}.jsonl` using fs.readFileSync.
    - Parse each line as JSON, filter by level/source/search if provided.
    - Return latest entries (reverse chronological), limited to `limit`.
    - If file doesn't exist, return empty array.

    **GET /api/system/logs/download** (app/src/app/api/system/logs/download/route.ts):
    - Accept query param `date` (default today).
    - Read raw file at `${LOG_DIR}/app-${date}.jsonl`.
    - Return as Response with headers: `Content-Type: application/x-ndjson`, `Content-Disposition: attachment; filename="app-${date}.jsonl"`.
    - If file doesn't exist, return 404.

    Also create the shared state module:
    **app/src/lib/testing-state.ts**:
    - Export interface `TestRunState { id: string; status: 'running' | 'passed' | 'failed'; output: string; startedAt: number }`.
    - Export `let currentRun: TestRunState | null = null`.
    - Export `function setCurrentRun(run: TestRunState | null)`.
    - Export `function getCurrentRun(): TestRunState | null`.
