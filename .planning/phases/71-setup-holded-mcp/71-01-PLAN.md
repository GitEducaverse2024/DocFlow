# Phase 71-01: Fork + Setup del Repositorio

## Goal

Clonar el repo `iamsamuelfraga/mcp-holded`, rebranding a DoCatFlow, eliminar complejidad multi-tenant, y agregar transporte HTTP con `StreamableHTTPServerTransport` en puerto 8766.

## Context

El repo base (`iamsamuelfraga/mcp-holded`, MIT) es un servidor MCP en TypeScript con 72 herramientas de facturacion. Usa transporte stdio solamente. Necesitamos adaptarlo al patron DoCatFlow (como se hizo con LinkedIn MCP en fase 47) y agregar transporte HTTP para acceso desde Docker.

Estructura base del repo:
```
src/
  index.ts              # Entry point, tool registration, stdio transport
  holded-client.ts      # HTTP client: auth, retry, backoff
  validation.ts         # Zod validation
  tools/                # 13 tool modules (contacts, documents, products, etc.)
  utils/
    rate-limiter.ts     # Per-tool rate limiting
    tenant-config.ts    # Multi-tenant (ELIMINAR)
    tenant-context.ts   # Multi-tenant (ELIMINAR)
```

Patron de referencia: `scripts/linkedin-mcp/setup.sh` (rebranding, eliminacion de atribucion).

### Task 3 — Agregar transporte HTTP con StreamableHTTPServerTransport

**Files:**
- `~/holded-mcp/src/index.ts` (MODIFICAR)
- `~/holded-mcp/package.json` (express ya agregado en Task 1)

**Action:**
1. En `src/index.ts`, agregar imports:
   ```typescript
   import { StreamableHTTPServerTransport } from '@modelcontextprotocol/sdk/server/streamableHttp.js';
   import express from 'express';
   ```
2. Despues de la creacion del servidor MCP y registro de herramientas, agregar bloque HTTP:
   ```typescript
   const app = express();
   app.use(express.json());

   app.all('/mcp', async (req, res) => {
     const transport = new StreamableHTTPServerTransport({ sessionIdGenerator: undefined });
     await server.connect(transport);
     await transport.handleRequest(req, res);
   });

   const PORT = parseInt(process.env.PORT || '8766');
   app.listen(PORT, '0.0.0.0', () => {
     console.error(`[holded-mcp] Servidor HTTP activo en http://0.0.0.0:${PORT}/mcp`);
   });
   ```
3. Mantener el transporte stdio como fallback (si se ejecuta sin `PORT`, usa stdio). Logica:
   ```typescript
   if (process.env.PORT || process.argv.includes('--http')) {
     // HTTP transport (arriba)
   } else {
     // stdio transport (codigo original)
   }
   ```
4. Ejecutar `cd ~/holded-mcp && npm install && npm run build`
5. Test rapido: `cd ~/holded-mcp && PORT=8766 timeout 5 node dist/index.js 2>&1 || true` -- debe imprimir "Servidor HTTP activo"

**Verification:**

- `grep -q "StreamableHTTPServerTransport" ~/holded-mcp/src/index.ts`
- `grep -q "express" ~/holded-mcp/package.json`
