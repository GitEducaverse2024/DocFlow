# Phase 71-04: Seed Conector + Health Check + UI en DoCatFlow

## Goal

Integrar el conector Holded MCP en DoCatFlow: seed en base de datos, health check en API, tarjeta en panel de sistema, dot en footer, y conocimiento en CatBot.

## Context

El patron esta establecido por LinkedIn MCP. Cada integracion MCP requiere:
1. Seed en `db.ts` (INSERT condicional del conector)
2. Health check en `api/health/route.ts` (POST initialize al MCP)
3. Tipo en `use-system-health.ts` (interfaz TypeScript)
4. Tarjeta en `system-health-panel.tsx` (Card con estado)
5. Dot en `footer.tsx` (indicador de estado en footer)
6. Conocimiento en `catbot-tools.ts` (FEATURE_KNOWLEDGE)

Este plan NO depende de que el servidor MCP este corriendo. Solo necesita saber las env vars (`HOLDED_MCP_URL`) y el patron de URLs. Puede ejecutarse en paralelo con Plan 03.

Patron existente (LinkedIn MCP):
- `db.ts` linea 1304-1343: seed `seed-linkedin-mcp`
- `health/route.ts` linea 67+: `linkedinMcpUrl`, checkService con POST initialize
- `use-system-health.ts` linea 11-17: `LinkedInMcpStatus` interface
- `system-health-panel.tsx` linea 91+: Card condicional `health.linkedin_mcp?.configured`
- `footer.tsx` linea 15: spread condicional en array de servicios
- `catbot-tools.ts` linea 277: entrada 'linkedin' en FEATURE_KNOWLEDGE

             params: {

               clientInfo: { name: 'docatflow-health', version: '1.0' }
             }

         });

       })
