# Phase 71-02: HTTP Client: Rate Limiting, Key Masking, Module URLs

## Goal

Adaptar `holded-client.ts` para cumplir requisitos DoCatFlow: delay minimo de 150ms entre peticiones, enmascaramiento de API key en logs, y URLs base configurables por modulo (invoicing, CRM, projects, team).

## Context

El `holded-client.ts` del repo base ya tiene retry con backoff exponencial y manejo de errores HTTP (429, 5xx). Las adaptaciones son incrementales sobre codigo funcional existente.

Modulos de la API de Holded con URLs base distintas:
- Invoicing: `https://api.holded.com/api/invoicing/v1/`
- CRM: `https://api.holded.com/api/crm/v1/`
- Projects: `https://api.holded.com/api/projects/v1/`
- Team: `https://api.holded.com/api/team/v1/`

El repo base solo cubre Invoicing. Fases 72-75 agregaran herramientas para CRM, Projects y Team, pero la infraestructura de URLs por modulo se prepara ahora.

**NOTA:** Este plan modifica archivos dentro de `~/holded-mcp/` que Plan 01 clona. Si se ejecutan en paralelo, Plan 01 debe clonar primero (Task 1 de Plan 01) antes de que este plan pueda editar. En la practica, Plan 01 Task 1 (clone) tarda ~10 segundos, asi que el executor de Plan 02 debe verificar que `~/holded-mcp/src/holded-client.ts` existe antes de editar.

## Tasks

### Task 1 — Delay minimo 150ms y enmascaramiento de API key

**Files:**
- `~/holded-mcp/src/holded-client.ts` (MODIFICAR)

**Action:**
1. Agregar propiedad de tracking del ultimo request al client:
   ```typescript
   private lastRequestTime: number = 0;
   private readonly MIN_DELAY_MS = 150;
   ```
2. En el metodo que hace las peticiones HTTP (probablemente `request()` o similar), agregar al inicio:
   ```typescript
   const now = Date.now();
   const elapsed = now - this.lastRequestTime;
   if (elapsed < this.MIN_DELAY_MS) {
     await new Promise(resolve => setTimeout(resolve, this.MIN_DELAY_MS - elapsed));
   }
   this.lastRequestTime = Date.now();
   ```
3. Para el enmascaramiento de la API key en logs:
   - Crear funcion helper: `private maskKey(key: string): string { return key.length > 8 ? key.slice(0, 4) + '****' + key.slice(-4) : '****'; }`
   - Buscar todos los `console.log`, `console.error`, o cualquier logger que imprima headers o la key
   - Reemplazar cualquier log que incluya el valor de la key con la version enmascarada
   - Verificar que el header `key` no aparezca en ningun log de request/response
4. Agregar un log al inicio del constructor o metodo init:
   ```typescript
   console.error(`[holded-client] API key: ${this.maskKey(this.apiKey)} | Min delay: ${this.MIN_DELAY_MS}ms`);
   ```

**Verification:**
- `grep -q "MIN_DELAY_MS" ~/holded-mcp/src/holded-client.ts`
- `grep -q "maskKey" ~/holded-mcp/src/holded-client.ts`
- `grep -n "console\.\(log\|error\)" ~/holded-mcp/src/holded-client.ts` -- ninguna linea imprime el valor completo de la API key

### Task 2 — URLs base configurables por modulo

**Files:**
- `~/holded-mcp/src/holded-client.ts` (MODIFICAR)

**Action:**
1. Definir tipo para modulos:
   ```typescript
   type HoldedModule = 'invoicing' | 'crm' | 'projects' | 'team';

   const MODULE_BASE_URLS: Record<HoldedModule, string> = {
     invoicing: 'https://api.holded.com/api/invoicing/v1',
     crm: 'https://api.holded.com/api/crm/v1',
     projects: 'https://api.holded.com/api/projects/v1',
     team: 'https://api.holded.com/api/team/v1',
   };
   ```
2. Modificar el metodo de peticion HTTP para aceptar un parametro `module` opcional:
   - Si se pasa `module`, usar `MODULE_BASE_URLS[module]` como base URL
   - Si no se pasa, usar el baseUrl por defecto (invoicing, para compatibilidad con herramientas existentes)
   - Ejemplo de firma: `async get(path: string, params?: Record<string, string>, module?: HoldedModule)`
3. Exportar el tipo `HoldedModule` para que las herramientas futuras (fases 72-75) puedan importarlo
4. Verificar que las herramientas existentes (contacts, documents, etc.) siguen funcionando sin cambios (no pasan `module`, asi que usan invoicing por defecto)
5. `cd ~/holded-mcp && npm run build` -- debe compilar sin errores

**Verification:**
- `cd ~/holded-mcp && npm run build` sin errores
- `grep -q "HoldedModule" ~/holded-mcp/src/holded-client.ts`
- `grep -q "MODULE_BASE_URLS" ~/holded-mcp/src/holded-client.ts`
- Las herramientas existentes no requieren cambios (son backward-compatible)
