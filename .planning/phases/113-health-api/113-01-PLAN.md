```typescript
// Types to export:
export interface ProviderHealth {
  provider: string;
  status: 'connected' | 'error';  // Map discovery's 'disconnected'/'no_key' to 'error'
  latency_ms: number | null;
  model_count: number;
  error: string | null;
}

export interface AliasHealth {
  alias: string;
  configured_model: string;       // model_key from alias row
  resolved_model: string | null;  // what resolveAlias() returned
  resolution_status: 'direct' | 'fallback' | 'error';
  error: string | null;
  latency_ms: number;
}

export interface HealthResult {
  providers: ProviderHealth[];
  aliases: AliasHealth[];
  checked_at: string;  // ISO timestamp
  cached: boolean;     // true if served from cache
}
```

**checkHealth implementation:**
1. Check cache key `health:result` — if hit and not force, return with `cached: true`
2. Call `getInventory(force)` from discovery to get fresh provider data
3. Map `inventory.providers` to `ProviderHealth[]` — map `disconnected`/`no_key` to `error` status
4. Call `getAllAliases({ active_only: true })` to get all active aliases
5. For each alias, call `resolveAlias(alias)` wrapped in try/catch:
   - Time the call with `Date.now()` delta
   - If the resolved model equals the configured model_key -> `direct`
   - If resolved model differs -> `fallback`
   - If throws -> `error` with error message, resolved_model = null
6. Build `HealthResult` with `checked_at: new Date().toISOString()`, `cached: false`
7. Store in cache with `cacheSet('health:result', result, 30_000)` (30s TTL)
8. Return result

Use `Promise.allSettled` to resolve all aliases in parallel (not sequential).

IMPORTANT: Use `process['env']` bracket notation per project convention. Follow existing service patterns (logger, withRetry where needed).

Create `app/src/lib/services/__tests__/health.test.ts` with jest mocks for discovery, alias-routing, and cache modules. Test the 6 behaviors listed above.
