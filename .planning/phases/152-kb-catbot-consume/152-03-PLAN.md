<objective>
Entregar el tercer mecanismo de consume: inyección pasiva de `_header.md` en el prompt de CatBot (KB-15) e inyección del campo `kb_entry` en los 5 tools canónicos de listado (KB-17). Reescribir `buildKnowledgeProtocol()` para que el LLM aprenda el orden `search_kb → get_kb_entry → query_knowledge → ...`. Validar end-to-end con fixture KB que los 5 list_* tools canónicos resuelven `kb_entry` correctamente.

**Scope sobre list_connectors:** KB-17 del contrato (REQUIREMENTS.md + ROADMAP.md Phase 152) enumera explícitamente 5 tools — `list_cat_paws`, `list_catbrains`, `list_skills`, `list_email_templates`, `canvas_list`. Un `list_connectors` genérico **no existe** en `catbot-tools.ts` (solo `list_email_connectors` para cuentas de email). Ese es el estado del codebase y queda deferred a una fase futura si se decide exponerlo.

Purpose: Plan 03 cierra los dos entregables que tocan el assembler y los list tools. Plans 01 y 02 fundaron el módulo + las tools nuevas; este plan integra esos cimientos en el hot-path del prompt y en el flujo de discovery que CatBot ejecuta a cada pregunta "lista X". Sin este plan, el kb_header no se inyecta y el LLM no ve el KB.

    ```typescript
    /**
     * Phase 152 KB-15 — Read `.docflow-kb/_header.md` and inject as P1 section.
     * Fresh read per request (no cache) — CONTEXT D1. File is <2KB (~33 lines); kb-sync.cjs --full-rebuild regenerates it and this fresh read guarantees CatBot sees changes immediately.
     * Graceful: returns '' on missing file (section not added).
     *
     * KB_ROOT resolution follows kb-index-cache.ts convention:
     *   - process['env']['KB_ROOT'] (bracket notation per MEMORY.md) if set
     *   - fallback: path.join(process.cwd(), '..', '.docflow-kb')
     *
     * Note on Docker CWD: in prod `cd /app && next start`, process.cwd()='/app' so fallback resolves to '/.docflow-kb'.
     * If Docker volume doesn't mount that path, Plan 04 must add the volume OR set KB_ROOT in env.
     */
    function buildKbHeader(): string {
      try {
        const kbRoot = process['env']['KB_ROOT'] || path.join(process.cwd(), '..', '.docflow-kb');
        const headerPath = path.join(kbRoot, '_header.md');
        const content = fs.readFileSync(headerPath, 'utf8');
        return content.trim();
      } catch {
        return '';
      }
    }
    ```
