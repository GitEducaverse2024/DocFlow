<objective>
Create the executeCatPaw() orchestration engine and integrate it into task-executor and canvas-executor.

Purpose: CatPaw is the unified entity replacing custom_agents and docs_workers. The execution engine loads a CatPaw with all its relations (CatBrains, connectors, skills), orchestrates RAG queries via executeCatBrain, invokes active connectors, builds the prompt with system_prompt + skill instructions + gathered context, calls LiteLLM, logs usage, and returns a structured CatPawOutput. Task and canvas executors must route through this engine when the agent_id points to a CatPaw.

Output: execute-catpaw.ts service, CatPawInput/CatPawOutput types, updated task-executor.ts and canvas-executor.ts
</objective>

**2. Create app/src/lib/services/execute-catpaw.ts** mirroring executeCatBrain pattern:

The function `executeCatPaw(pawId: string, input: CatPawInput): Promise<CatPawOutput>` must:

a) **Load CatPaw with relations** using a single query + 3 relation queries:
   - Main: `SELECT * FROM cat_paws WHERE id = ? AND is_active = 1`
   - CatBrains: `SELECT cpc.*, c.name as catbrain_name FROM cat_paw_catbrains cpc LEFT JOIN catbrains c ON c.id = cpc.catbrain_id WHERE cpc.paw_id = ? ORDER BY cpc.priority DESC`
   - Connectors: `SELECT cpc.*, c.name as connector_name, c.type as connector_type, c.config FROM cat_paw_connectors cpc LEFT JOIN connectors c ON c.id = cpc.connector_id WHERE cpc.paw_id = ? AND cpc.is_active = 1`
   - Skills: `SELECT s.name, s.instructions FROM cat_paw_skills cps JOIN skills s ON s.id = cps.skill_id WHERE cps.paw_id = ?`
   - Throw Error('CatPaw no encontrado o inactivo') if not found

b) **Query linked CatBrains** via executeCatBrain() (imported from execute-catbrain.ts — DO NOT modify that file):
   - For each linked catbrain, call `executeCatBrain(catbrainId, { query: input.query, context: input.context, mode: catbrain.query_mode })`
   - Collect answers into `catbrainContext` string, sources into `allSources` array, connector_data into `allConnectorData` array
   - Use withRetry wrapper (import from `@/lib/services/litellm` — check if withRetry is exported, otherwise wrap in try/catch with 1 retry)
   - Log errors but don't fail the whole execution if one CatBrain fails

c) **Invoke active connectors** — for each connector in the loaded relations:
   - Parse connector config JSON
   - Build payload: `{ paw_id: pawId, paw_name: paw.name, query: input.query, context: input.context }`
   - Call connector URL with fetch (method from config, timeout from config defaulting to 30s, AbortController)
   - Collect results; log errors but don't fail

d) **Build messages array**:
   - System message: Start with paw.system_prompt (or default `"Eres ${paw.name}, un asistente experto."`)
   - If paw.tone: append `"\nTono: ${paw.tone}"`
   - If skills loaded: append `"\n--- SKILLS ---\n${skills.map(s => '### ' + s.name + '\n' + s.instructions).join('\n\n')}\n--- FIN SKILLS ---"`
   - If catbrainContext: append `"\n--- CONOCIMIENTO CATBRAINS ---\n${catbrainContext}\n--- FIN CONOCIMIENTO CATBRAINS ---"`
   - If connector results: append `"\n--- DATOS DE CONECTORES ---\n${formattedConnectorResults}\n--- FIN DATOS CONECTORES ---"`
   - If paw.mode === 'processor' and paw.processing_instructions: append `"\n--- INSTRUCCIONES DE PROCESAMIENTO ---\n${paw.processing_instructions}\nFormato de salida: ${paw.output_format}\n--- FIN INSTRUCCIONES ---"`
   - User message: Build from input.query, prepend input.context if present (same pattern as executeCatBrain), prepend input.document_content if present (for processor mode)

e) **Call LiteLLM** with withRetry pattern:
   - `const model = await litellm.resolveModel(paw.model || process['env']['CHAT_MODEL'] || 'gemini-main')`
   - `const litellmUrl = process['env']['LITELLM_URL'] || 'http://localhost:4000'`
   - `const litellmKey = process['env']['LITELLM_API_KEY'] || 'sk-antigravity-gateway'`
   - POST to `${litellmUrl}/v1/chat/completions` with `{ model, messages, temperature: paw.temperature, max_tokens: paw.max_tokens }`
   - Parse response, extract answer and usage

f) **Log usage** via logUsage():
   ```typescript
   logUsage({
     event_type: 'chat',
     agent_id: pawId,       // paw_id stored in agent_id column
     model,
     input_tokens: usage.prompt_tokens || 0,
     output_tokens: usage.completion_tokens || 0,
     total_tokens: usage.total_tokens || 0,
     duration_ms: durationMs,
     status: 'success',
     metadata: { paw_name: paw.name, mode: paw.mode },
   });
   ```

g) **Update times_used**: `UPDATE cat_paws SET times_used = times_used + 1 WHERE id = ?`

h) **Return CatPawOutput** with all fields populated.

**Critical patterns to follow:**
- All `process.env` access via bracket notation: `process['env']['VAR']`
- Import db from `@/lib/db`, logger from `@/lib/logger`
- Use `logger.info('cat-paws', ...)` and `logger.error('cat-paws', ...)` — the 'cat-paws' LogSource was added in Phase 43
- DO NOT import from or modify execute-catbrain.ts internals — only call the exported `executeCatBrain` function
- Wrap external calls (executeCatBrain, connector fetch, LiteLLM fetch) in try/catch — individual failures should not crash the whole execution
