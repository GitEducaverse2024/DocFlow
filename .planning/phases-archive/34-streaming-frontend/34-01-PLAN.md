**Create `app/src/hooks/use-sse-stream.ts`:**

A shared React hook that manages SSE consumption from POST endpoints with requestAnimationFrame batching.

Interface:
```typescript
interface UseSSEStreamOptions {
  onToken: (token: string) => void;
  onToolCallStart?: (data: { name: string }) => void;
  onToolCallResult?: (data: { name: string; result: unknown }) => void;
  onStage?: (data: { stage: string; message: string }) => void;
  onDone: (data: Record<string, unknown>) => void;
  onError: (error: Error) => void;
}

interface UseSSEStreamReturn {
  start: (url: string, body: Record<string, unknown>) => void;
  stop: () => void;
  isStreaming: boolean;
}
```

Implementation details:
1. `start(url, body)` creates a new AbortController, stores it in a ref. Calls `fetch(url, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ ...body, stream: true }), signal: controller.signal })`.
2. Reads response body via `response.body!.getReader()` + `TextDecoder`.
3. Keeps a `buffer` string. Splits on `\n`, keeps last incomplete segment. Parses `event:` and `data:` lines. Dispatches to appropriate callback based on event name (token, tool_call_start, tool_call_result, stage, done, error).
4. **requestAnimationFrame batching for onToken:** Do NOT call onToken directly for each token. Instead, accumulate tokens in a local string variable. On each token arrival, if no rAF is pending, schedule one that flushes the accumulated tokens via a single onToken call. This limits state updates to ~60/second max. Clean up pending rAF in stop() and on unmount.
5. `stop()` calls `abortController.abort()`. In the reader loop catch block, check `if (error.name === 'AbortError') return` — do NOT call onError for user-initiated stops.
6. `isStreaming` state: set true when start() is called, set false on done/error/abort.
7. On component unmount (useEffect cleanup), call stop() to prevent orphaned streams.
8. If fetch response is not ok (status >= 400), read the body as JSON and call onError with the error message.

**Add to `app/src/app/globals.css`** (after existing animations, before the closing content):

```css
/* Streaming cursor */
@keyframes streaming-blink {
  0%, 100% { opacity: 1; }
  50% { opacity: 0; }
}

```

The cursor uses CSS ::after pseudo-element — NOT a character appended to content string. This prevents cursor from interfering with markdown rendering (appearing inside code blocks, etc.).

**Hook setup:**
```typescript
const { start, stop, isStreaming } = useSSEStream({
  onToken: (token) => {
    setStreamingContent(prev => prev + token);
  },
  onDone: (data) => {
    // Convert streaming content to a final message
    setMessages(prev => [...prev, {
      role: 'bot',
      content: streamingContentRef.current,  // Use ref for latest value
      sources: (data as any).sources
    }]);
    setStreamingContent('');
    streamingContentRef.current = '';
  },
  onError: (error) => {
    // If there was partial content, save it as a message
    if (streamingContentRef.current) {
      setMessages(prev => [...prev, {
        role: 'bot',
        content: streamingContentRef.current
      }]);
    }
    setStreamingContent('');
    streamingContentRef.current = '';
    console.error('Stream error:', error);
  },
});
```

Add `const streamingContentRef = useRef('')` and keep it synced: in onToken callback, also update `streamingContentRef.current`. This avoids stale closure issues in onDone.
