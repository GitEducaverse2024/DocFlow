1. Create `app/src/hooks/use-notifications.ts`:
   - `useNotifications(pollInterval = 15000)` hook modeled on `useSystemHealth`
   - State: `unreadCount` (number), `recent` (Notification[]), `isLoading` (boolean)
   - `fetchCount()`: GET `/api/notifications/count` with `{ cache: 'no-store' }`, update `unreadCount`
   - `fetchRecent()`: GET `/api/notifications?limit=20` with `{ cache: 'no-store' }`, update `recent`
   - `markAllRead()`: POST `/api/notifications/read-all`, then call `fetchCount()` to refresh badge
   - `markOneRead(id: string)`: PATCH `/api/notifications/${id}/read` with method PATCH. After the PATCH succeeds, apply an optimistic update: `setUnreadCount(prev => Math.max(0, prev - 1))`. Also update the `recent` array locally to set `read: 1` on the matching notification so the UI reflects the change immediately without waiting for the next poll or refetch.
   - useEffect: on mount call `fetchCount()`, set interval every `pollInterval` ms, cleanup on unmount
   - Do NOT auto-fetch the full recent list on interval -- only fetch count (lightweight). Fetch recent list only when explicitly called (when popover opens).
   - Return: `{ unreadCount, recent, isLoading, fetchCount, fetchRecent, markAllRead, markOneRead }`
   - Define a frontend `Notification` interface matching the API response shape
