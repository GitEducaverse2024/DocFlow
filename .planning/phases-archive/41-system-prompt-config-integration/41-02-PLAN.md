Create `app/src/components/catbrains/config-panel.tsx` — a "use client" component.

Props: `{ catbrain: Project; onCatBrainUpdate: () => void; onDelete: () => void }`

Sections (vertical scroll, same zinc-950 dark theme as existing panels):

1. **Informacion basica** section:
   - Nombre: Input field, required
   - Descripcion: Textarea, 2 rows, placeholder "Descripcion del CatBrain..."

2. **System Prompt** section:
   - Textarea with `min-h-[120px]` and `resize-y` to allow vertical expansion
   - Placeholder: "Instrucciones de personalidad para el CatBrain... (ej: Eres un experto en...)"
   - Small helper text below: "Este prompt se inyecta en cada interaccion LLM del CatBrain"
   - Initialize from `catbrain.system_prompt || ''`

3. **Modelo LLM** section:
   - Label: "Modelo por defecto"
   - Select dropdown populated by fetching GET `/api/models` on mount
   - First option: "Automatico (modelo del sistema)" with value `''`
   - Rest: model names from the API response
   - Initialize from `catbrain.default_model || ''`

4. **MCP Endpoint** section:
   - Toggle switch (custom div with onClick, same pattern as existing toggles in the codebase) for `mcp_enabled`
   - When enabled, show a read-only input with the URL: `http://${window.location.hostname}:3500/api/mcp/${catbrain.id}`
   - Next to the URL: a "Copiar" button that copies to clipboard via `navigator.clipboard.writeText()` and shows toast "URL copiada"
   - When disabled, show text "Endpoint MCP desactivado"

5. **Guardar** button — sticky at bottom:
   - Calls PATCH `/api/catbrains/${catbrain.id}` with `{ name, description, system_prompt, default_model, mcp_enabled }`
   - Toast on success/error via sonner
   - Disabled while saving or if name is empty

6. **Zona peligrosa** section (border-red styling):
   - "Eliminar CatBrain" button that calls `onDelete()`

State management:
- `useState` for each field (name, description, systemPrompt, defaultModel, mcpEnabled)
- `useState<string[]>` for models, fetched from `/api/models` on mount
- `useState` for saving boolean

Style: Use existing shadcn components (Input, Textarea, Button, Label) and Tailwind classes matching the zinc-950/zinc-900/zinc-800 theme. All text in Spanish.
