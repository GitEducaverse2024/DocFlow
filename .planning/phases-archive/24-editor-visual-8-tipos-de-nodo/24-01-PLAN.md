   ```css
     --xy-edge-stroke: theme(colors.violet.500);
     --xy-edge-stroke-width: 2;

   ```

4. Create `app/src/app/canvas/[id]/page.tsx` — server-compatible page shell:
   - Import `dynamic` from `next/dynamic`
   - Dynamically import `@/components/canvas/canvas-editor` with `{ ssr: false }` and a loading fallback div with `className="h-screen bg-zinc-950 animate-pulse"`
   - Extract `params.id` and pass as `canvasId` prop to CanvasEditor
   - Add breadcrumb-compatible title metadata if desired

5. Create `app/src/components/canvas/canvas-editor.tsx` — the core editor:
   - `"use client"` directive at top
   - Import from `@xyflow/react`: ReactFlow, ReactFlowProvider, Background, BackgroundVariant, MiniMap, Controls, useNodesState, useEdgesState, useReactFlow, addEdge, Connection, Node, Edge, useViewport
   - Define `NODE_TYPES` as module-level constant (empty object for now — Plan 02 will populate). This is CRITICAL — never inside component body (Pitfall 4).
   - Define `getDefaultNodeData(nodeType: string)` function returning default data per type (use the exact defaults from RESEARCH.md section "Default Node Data per Type")
   - Export `CanvasEditor` component that renders `<ReactFlowProvider><CanvasShell canvasId={canvasId} /></ReactFlowProvider>` — the provider must wrap EVERYTHING including toolbar and palette (Pitfall 5)
   - `CanvasShell` inner component:
     - `useNodesState([])` and `useEdgesState([])`
     - `useReactFlow()` for fitView, toObject, screenToFlowPosition
     - `useEffect` on mount: fetch `/api/canvas/${canvasId}`, parse flow_data JSON, setNodes/setEdges, setTimeout fitView 50ms
     - `isValidConnection` callback with DFS cycle check (see RESEARCH.md Pattern 5) — checks if adding the edge from source to target would create a cycle by walking outgoing edges from target
     - `onConnect` callback: `setEdges(eds => addEdge(connection, eds))`
     - `onDrop` handler: reads `application/reactflow` data transfer, uses `screenToFlowPosition` for position, creates new node with `generateId()` from lib/utils, calls `setNodes(prev => [...prev, newNode])`
     - `onDragOver` handler: `event.preventDefault(); event.dataTransfer.dropEffect = 'move'`
     - Render layout: flex column, full height
       - `<CanvasToolbar canvasId={canvasId} />` at top
       - flex row below: `<NodePalette />` (left) + canvas container (flex-1)
       - Canvas container div: `className="flex-1 h-[calc(100vh-64px)]"` (Pitfall 2)
       - `<ReactFlow>` with props: nodes, edges, onNodesChange, onEdgesChange, onConnect, onDrop, onDragOver, nodeTypes={NODE_TYPES}, snapToGrid, snapGrid={[20,20]}, isValidConnection, deleteKeyCode={['Delete','Backspace']}, selectionOnDrag, multiSelectionKeyCode="Shift", fitView
       - Inside ReactFlow: `<Background variant={BackgroundVariant.Dots} gap={20} size={1.5} color="#3f3f46" />`, `<MiniMap className="!bg-zinc-900" />`, `<Controls className="!bg-zinc-900 !border-zinc-700" />`
     - For minimap nodeColor, create a `getMiniMapNodeColor` function that returns colors per node type (emerald for start/output, violet for agent, blue for project, orange for connector, amber for checkpoint, cyan for merge, yellow for condition)
