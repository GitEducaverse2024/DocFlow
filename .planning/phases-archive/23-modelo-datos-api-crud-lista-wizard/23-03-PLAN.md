  <action>
Create `app/src/components/canvas/canvas-wizard.tsx` — "use client" component.

Props:
```typescript
interface CanvasWizardProps {
  open: boolean;
  onClose: () => void;
  onCreated: (id: string) => void;
  initialMode?: 'agents' | 'projects' | 'mixed' | 'template'; // optional preset from template "Usar" button
}
```

Use shadcn Dialog, DialogContent, DialogHeader, DialogTitle, DialogDescription. Import Button, Input, Textarea, Badge from shadcn.

**Internal state:**
- step: 1 | 2
- mode: 'agents' | 'projects' | 'mixed' | 'template' | null
- selectedTemplateId: string | null
- name: string
- description: string
- emoji: string (default '🔷')
- tags: string (comma-separated text input)
- loading: boolean
- templates: array (fetched when mode === 'template')

**Step 1 — Seleccion de tipo (WIZ-01):**
4 clickable cards in a 2x2 grid:
1. **Agentes** — icon Bot, "Workflows de agentes IA con checkpoints y condiciones", color violet
2. **Proyectos** — icon FolderKanban, "Pipelines de procesamiento de proyectos", color blue
3. **Mixto** — icon Layers, "Combina agentes y proyectos en un flujo unificado", color emerald
4. **Desde Plantilla** — icon FileText, "Comienza desde una plantilla predefinida", color amber

Each card: bg-zinc-900 border border-zinc-800, hover:border-{color}-500/50. On click: set mode, advance to step 2. If initialMode is set, auto-advance to step 2.

**Step 2 — Detalles (WIZ-02, WIZ-03):**

If mode === 'template' (WIZ-03):
- First show a template list fetched from GET /api/canvas/templates
- Each template as a selectable card with preview_svg, name, description, times_used count
- If no templates available, show message "No hay plantillas disponibles. Las plantillas se agregaran en futuras actualizaciones."
- After selecting a template, show the name/description/emoji/tags form below

For all modes (including after template selection):
- Input: "Nombre del canvas" (required, placeholder "Mi workflow")
- Textarea: "Descripcion (opcional)" (placeholder "Describe el proposito de este workflow...")
- Emoji selector: a row of 12 preset emojis relevant to workflows: 🔷 🚀 📊 🤖 📝 ⚡ 🎯 🔄 📋 💡 🛠️ ⭐ — clicking one sets the emoji state. Also show a small text input allowing custom emoji paste.
- Tags input: "Etiquetas (separadas por coma)" — simple text input, placeholder "ej: produccion, marketing"
- "Crear y abrir editor" button (disabled when name is empty or loading)
- "Atras" button to go back to step 1

**Submit logic:**
- If mode !== 'template': POST /api/canvas with { name, description, emoji, mode, tags: tags.split(',').map(t => t.trim()).filter(Boolean) }
- If mode === 'template': POST /api/canvas/from-template with { templateId: selectedTemplateId, name, description, emoji, tags: ... }
- On success: call onCreated(response.id) which triggers router.push in parent
- On error: show toast via sonner

**Dialog styling:**
- DialogContent: max-w-lg for step 1, max-w-lg for step 2
- Smooth transition between steps (no animation library needed, just conditional rendering)
- Close dialog resets all state to initial

**Reset on close:**
When the dialog closes (onClose or after creation), reset step to 1, mode to null, all form fields to defaults.
