  <name>Task 1: Create useTestRunner hook with polling</name>
  <files>app/src/hooks/use-test-runner.ts</files>
  <action>
    Create a custom hook `useTestRunner` that manages test execution state and data fetching.

    **State:**
    - `runs: TestRun[]` — historical test runs from GET /api/testing/results
    - `latestRun: TestRun | null` — first item from runs (most recent)
    - `isRunning: boolean` — true while a test execution is in progress
    - `runOutput: string` — stdout from current run (from status endpoint)
    - `loading: boolean` — true during initial data fetch

    **Interface types (define in the hook file or a shared types file):**
    ```typescript
    interface TestResult { title: string; file: string; status: 'passed' | 'failed' | 'skipped'; duration: number; error?: string }
    interface TestRun { id: string; type: string; section: string | null; status: string; total: number; passed: number; failed: number; skipped: number; duration_seconds: number; results_json: TestResult[]; created_at: string }
    ```

    **Functions:**
    - `fetchResults()` — GET /api/testing/results?limit=10, parse response, set runs and latestRun
    - `runTests(section?: string)` — POST /api/testing/run with optional section body, set isRunning=true, start polling
    - Polling: useEffect that runs when isRunning is true. setInterval every 2000ms calling GET /api/testing/status. When status is not 'running', clear interval, set isRunning=false, call fetchResults() to refresh.

    **On mount:** Call fetchResults() once.

    **Return:** `{ runs, latestRun, isRunning, runOutput, loading, runTests, fetchResults }`
  </action>

  </verify>
  <done>useTestRunner hook compiles. Exports runs, latestRun, isRunning, runTests function. Polls /api/testing/status every 2s when running.</done>
