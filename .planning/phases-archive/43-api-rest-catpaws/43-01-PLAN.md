  <name>Task 1: GET list + POST create at /api/cat-paws</name>
  <files>app/src/app/api/cat-paws/route.ts</files>
  <action>
Create `app/src/app/api/cat-paws/route.ts` with `export const dynamic = 'force-dynamic'`.

**GET handler** — List CatPaws with optional filters and relation counts:
- Read query params from `request.url`: `mode` (chat/processor/hybrid), `department` (string to match in department_tags JSON), `active` (0 or 1)
- Build SQL dynamically: base query is `SELECT cp.*, (SELECT COUNT(*) FROM cat_paw_skills WHERE paw_id = cp.id) as skills_count, (SELECT COUNT(*) FROM cat_paw_catbrains WHERE paw_id = cp.id) as catbrains_count, (SELECT COUNT(*) FROM cat_paw_connectors WHERE paw_id = cp.id) as connectors_count, (SELECT COUNT(*) FROM cat_paw_agents WHERE paw_id = cp.id) as agents_count FROM cat_paws cp`
- Append WHERE clauses: if `mode` param, add `cp.mode = ?`; if `active` param, add `cp.is_active = ?`; if `department` param, add `cp.department_tags LIKE ?` (use `%"value"%` pattern for JSON array search)
- Order by `cp.updated_at DESC`
- Return `NextResponse.json(rows)`
- Wrap in try/catch, log errors with `logger.error('cat-paws', ...)`

**POST handler** — Create new CatPaw:
- Parse body JSON. Required field: `name` (return 400 if missing)
- Generate `id` with `generateId()` from `@/lib/utils`
- Set defaults: `avatar_emoji` = body.avatar_emoji || '🐾', `avatar_color` = body.avatar_color || '#8B6D8B', `tone` = body.tone || 'professional', `mode` = body.mode || 'chat', `model` = body.model || 'gemini-main', `temperature` = body.temperature ?? 0.7, `max_tokens` = body.max_tokens ?? 4096, `output_format` = body.output_format || 'md', `is_active` = body.is_active ?? 1
- `department_tags`: if body.department_tags is an array, JSON.stringify it; if string, use as-is; else null
- `now = new Date().toISOString()` for created_at and updated_at
- INSERT INTO cat_paws with all fields (id, name, description, avatar_emoji, avatar_color, department_tags, system_prompt, tone, mode, model, temperature, max_tokens, processing_instructions, output_format, is_active, times_used=0, created_at, updated_at). Nullable fields: description, system_prompt, processing_instructions, openclaw_id, openclaw_synced_at
- SELECT the created row back and return with `NextResponse.json(row, { status: 201 })`
- Log with `logger.info('cat-paws', 'CatPaw creado', { pawId: id, name })`

Imports: `NextResponse` from next/server, `db` from @/lib/db, `generateId` from @/lib/utils, `logger` from @/lib/logger.

<task type="auto">
  <name>Task 2: GET detail + PATCH update + DELETE at /api/cat-paws/[id]</name>
  <files>app/src/app/api/cat-paws/[id]/route.ts</files>
  <action>
Create `app/src/app/api/cat-paws/[id]/route.ts` with `export const dynamic = 'force-dynamic'`.

All handlers receive `{ params }: { params: Promise<{ id: string }> }` and do `const { id } = await params;` (Next.js 15 async params pattern, same as catbrains).

**GET handler** — Detail with full relations:
- First query: `SELECT * FROM cat_paws WHERE id = ?` — if not found, return 404
- Then load relations in parallel (4 separate queries, all synchronous in better-sqlite3):
  - `SELECT cpc.*, c.name as catbrain_name FROM cat_paw_catbrains cpc LEFT JOIN catbrains c ON c.id = cpc.catbrain_id WHERE cpc.paw_id = ?` — join catbrains table to get name
  - `SELECT cpc.*, cn.name as connector_name, cn.type as connector_type FROM cat_paw_connectors cpc LEFT JOIN connectors cn ON cn.id = cpc.connector_id WHERE cpc.paw_id = ?` — join connectors table to get name+type
  - `SELECT cpa.*, cp2.name as target_name, cp2.avatar_emoji as target_emoji FROM cat_paw_agents cpa LEFT JOIN cat_paws cp2 ON cp2.id = cpa.target_paw_id WHERE cpa.paw_id = ?` — self-join for target paw info
  - `SELECT cps.*, s.name as skill_name FROM cat_paw_skills cps LEFT JOIN skills s ON s.id = cps.skill_id WHERE cps.paw_id = ?` — join skills table
- Return `NextResponse.json({ ...paw, catbrains, connectors, agents, skills })`

**PATCH handler** — Partial update:
- Check paw exists (404 if not)
- Updatable fields: name, description, avatar_emoji, avatar_color, department_tags, system_prompt, tone, mode, model, temperature, max_tokens, processing_instructions, output_format, is_active, openclaw_id, openclaw_synced_at
- Use same dynamic update pattern as catbrains: for each field, `if (field !== undefined) { updates.push('field = ?'); values.push(field); }`
- Special handling for `department_tags`: if value is an array, JSON.stringify it before storing
- Always append `updated_at = ?` with `new Date().toISOString()`
- If no updates, return current paw as-is
- Execute UPDATE, SELECT updated row, return JSON
- Log with `logger.info('cat-paws', 'CatPaw actualizado', { pawId: id })`

**DELETE handler** — Remove with cascade:
- Check paw exists (404 if not)
- Execute `DELETE FROM cat_paws WHERE id = ?` — CASCADE constraints in the relation tables handle cleanup automatically (ON DELETE CASCADE was set in Phase 42 table definitions)
- Return `NextResponse.json({ success: true })`
- Log with `logger.info('cat-paws', 'CatPaw eliminado', { pawId: id })`

Imports: `NextResponse` from next/server, `db` from @/lib/db, `logger` from @/lib/logger.
