Create `app/src/app/api/cat-paws/[id]/skills/route.ts` (~50 lines).

- `export const dynamic = 'force-dynamic'`
- Import `getDb` from `@/lib/db`

**POST handler** — Add skill to CatPaw:
- Extract `id` from params (paw_id)
- Parse body: `{ skill_id: string }`
- Validate: paw exists (`SELECT id FROM cat_paws WHERE id = ?`), skill exists (`SELECT id FROM skills WHERE id = ?`)
- If either missing, return 404 with error message
- INSERT OR IGNORE INTO cat_paw_skills (paw_id, skill_id) VALUES (?, ?)
- Return 201 `{ paw_id, skill_id }`

**DELETE handler** — Remove skill from CatPaw:
- Extract `id` from params (paw_id)
- Parse body: `{ skill_id: string }`
- DELETE FROM cat_paw_skills WHERE paw_id = ? AND skill_id = ?
- If changes === 0, return 404 "Skill no vinculado"
- Return 200 `{ deleted: true }`

**GET handler** — List skills for CatPaw:
- Extract `id` from params
- `SELECT s.* FROM skills s JOIN cat_paw_skills cps ON s.id = cps.skill_id WHERE cps.paw_id = ?`
- Return 200 with skills array

Follow exact patterns from existing relation endpoints (e.g., `/api/cat-paws/[id]/catbrains/route.ts`). Use try/catch with 500 error responses.

Create "use client" page at `app/src/app/agents/new/page.tsx`.

**Wizard structure** — 4 steps displayed with a horizontal stepper (numbered circles with connecting lines, similar to PipelineNav pattern but simpler):

**Step 1 — Identidad:**
- name (required, Input)
- avatar_emoji (Input, default "🐱")
- avatar_color (color picker or preset palette of 6 colors: violet, teal, amber, rose, blue, emerald — as clickable circles)
- department_tags (Input, comma-separated, hint: "ventas, soporte, marketing")
- mode selector: 3 cards (Chat, Procesador, Hibrido) with icons and descriptions. Chat = "Responde preguntas y conversa", Procesador = "Procesa documentos automaticamente", Hibrido = "Chat + procesamiento". Each card highlights with its mode color on selection.
- description (Textarea, optional)

**Step 2 — Personalidad:**
- system_prompt (Textarea, large, placeholder: "Eres un asistente que...")
- tone (Select: profesional, casual, tecnico, creativo, formal)
- model (Input, default "gpt-4o-mini")
- temperature (Slider 0-2, default 0.7, show value)
- max_tokens (Input number, default 2048)
- If mode is processor or hybrid, also show:
  - processing_instructions (Textarea)
  - output_format (Select: markdown, json, text, csv)

**Step 3 — Skills:**
- Fetch skills from `GET /api/skills` on mount
- Multi-select checklist of available skills (checkboxes with skill name)
- Selected skills stored in local state as string[]

**Step 4 — Conexiones:**
- Three sections collapsible (CatBrains, Conectores, Agentes)
- CatBrains: fetch from `GET /api/catbrains?limit=100`, show checklist. For each selected, show query_mode select (rag/connector/both) and priority input (number, default 1).
- Conectores: fetch from `GET /api/connectors`, show checklist of active ones. For each selected, show usage_hint input.
- Agentes: fetch from `GET /api/cat-paws`, show checklist (exclude self — but this is new, so no self). For each selected, show relationship select (collaborator/delegate/supervisor).

**Navigation:** Back/Next buttons. Step 1 requires name and mode. Final step shows "Crear CatPaw" button.

**Submit flow:**
1. POST /api/cat-paws with identity + personality fields
2. For each selected skill: POST /api/cat-paws/[newId]/skills with `{ skill_id }` (uses the skills endpoint from Task 1)
3. For each catbrain: POST /api/cat-paws/[newId]/catbrains
4. For each connector: POST /api/cat-paws/[newId]/connectors
5. For each agent: POST /api/cat-paws/[newId]/agents
6. On success, toast "CatPaw creado" and router.push(`/agents/${newId}`)
7. On error, toast error message

**Style:** bg-zinc-950, cards bg-zinc-900 border-zinc-800, buttons gradient violet. All text in Spanish.

**Chat API route** (`app/src/app/api/cat-paws/[id]/chat/route.ts`):
- `export const dynamic = 'force-dynamic'` and `export const runtime = 'nodejs'`
- POST handler: receives `{ message, stream }` from body
- Load CatPaw from DB: `SELECT * FROM cat_paws WHERE id = ?`
- If !paw or mode === 'processor', return 400 "Este CatPaw no soporta chat"
- Import and call `executeCatPaw` from `@/lib/services/execute-catpaw` with `{ query: message }`
- If `stream` is truthy: use `createSSEStream` and `sseHeaders` from stream-utils. Inside handler: call executeCatPaw, send 'token' events as answer streams (NOTE: executeCatPaw returns CatPawOutput not a stream — so for streaming, build messages array same as executeCatPaw does internally, then call `streamLiteLLM` directly with the same system prompt / context assembly). Pattern: load paw + relations, build system prompt, query catbrains for context, assemble messages, then streamLiteLLM with onToken sending SSE events. Send 'sources' event at end if RAG sources found. Send 'done' event with usage. Return `new Response(stream, { headers: sseHeaders })`.
- If not streaming: call executeCatPaw normally, return JSON response.
- Follow exact patterns from catbrains/[id]/chat/route.ts for SSE structure.

**Detail page** (`app/src/app/agents/[id]/page.tsx`):
- "use client" page

**Tab 1 — Identidad:**

- "Eliminar" button (red, with confirm dialog) → DELETE /api/cat-paws/[id], redirect to /agents

**Tab 2 — Conexiones (UI-05):**
- 3 sections: CatBrains vinculados, Conectores vinculados, Agentes vinculados
- Each section: list of linked items with unlink button (trash icon, calls DELETE endpoint)
- Each section: "Vincular" button that opens a dialog/sheet to select from available items and set config (query_mode for catbrains, usage_hint for connectors, relationship for agents)

**Tab 3 — Skills:**
