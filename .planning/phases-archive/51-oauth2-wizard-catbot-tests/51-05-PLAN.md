Read the existing CONNECTORS.md and add a comprehensive "Conector Gmail" section (DOC-01). Include:

**Modos de Autenticacion:**
- App Password (Personal): Requires 2FA enabled, App Password from myaccount.google.com/apppasswords, uses smtp.gmail.com:587
- App Password (Workspace): Same but uses smtp-relay.gmail.com:587, requires admin to enable SMTP relay
- OAuth2 (Workspace): Requires Google Cloud Console project, Gmail API enabled, OAuth2 credentials, consent screen configured

**Configuracion desde la UI:**
- Wizard de 4 pasos: seleccion de cuenta, credenciales, test, confirmacion
- Gmail type aparece con badge esmeralda en /conectores

**Uso desde Canvas/Tareas/CatBot:**
- Canvas: agregar nodo Conector Gmail al final del flujo, output del nodo previo se parsea como email (JSON con to/subject/body, o texto plano)
- Tareas: seleccionar conector Gmail en un paso de tarea
- CatBot: "Envia un email usando [nombre del conector]" - CatBot confirma antes de enviar

**Troubleshooting (8 errores comunes):**
1. "Credenciales invalidas" - App Password incorrecta o expirada, verificar en myaccount.google.com
2. "No se pudo conectar al servidor SMTP" - Firewall bloqueando puerto 587/465, verificar red
3. "Tiempo de espera agotado" - Red lenta o SMTP bloqueado por ISP
4. "Error de certificado SSL" - Proxy interceptando TLS, verificar configuracion de red
5. "No se obtuvo refresh_token" - Revocar acceso en myaccount.google.com/permissions y reintentar OAuth2
6. "Rate limit exceeded" - Limite diario: 500 (personal), 2000 (workspace), 10000 (smtp-relay)
7. "Invalid grant" - Codigo de autorizacion expirado o ya usado, generar nuevo
8. "Access denied" - Gmail API no habilitada en Google Cloud Console, o consent screen pendiente de verificacion

**Arquitectura:**
- Credenciales cifradas con AES-256-GCM (CONNECTOR_SECRET)
- Anti-spam: delay 1s entre envios del mismo conector
- OAuth2 usa refresh_token para obtener access_token fresco en cada envio

Also note OAUTH-04 and OAUTH-05 are documented here as reference for the wizard's inline instructions (the actual UI implementation is in plan 03, but the content specification lives in docs).
